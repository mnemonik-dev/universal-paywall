# Task 1: Monorepo scaffolding (ESM-only)

Перед выполнением задачи загрузи:

## Description

Поднять с нуля корневой npm workspace монорепо `universal-paywall` и каркас пакета `packages/middleware` как ESM-only TypeScript-проекта, готового под последующие волны (контракты, middleware-модули, тесты, деплой). Сейчас в репозитории нет ни `package.json`, ни `packages/`, ни линтера, ни pre-commit — только `.gitignore`, `CLAUDE.md` и папка `work/`. Эта задача создаёт фундамент, на который опираются все остальные задачи feature x402-agent-payment.

Ключевые архитектурные ограничения (tech-spec, Solution + Acceptance Criteria):
- `@universal-paywall/middleware` публикуется как **ESM-only** (`"type": "module"`, exports map без CJS-ветки), `engines.node: ">=20"`.
- Сборка через **tsup** (D7 косвенно — TypeScript-native стек; tsup явно указан в Dependencies → "New (middleware)").
- TypeScript strict-режим.
- `npm install && npm run lint && npm run build --workspace=packages/middleware` должен проходить.
- gitleaks как pre-commit hook (см. patterns.md "Security Gates → Pre-commit: gitleaks").
- ESLint + Prettier — pre-commit (TypeScript typecheck + ESLint per patterns.md).

Скрипты вроде `tsx`, `vitest`, `viem`, `ajv` ставятся в **следующих** волнах (Tasks 6+). Здесь — только инфраструктурный минимум, достаточный чтобы пакет собирался пустой `src/index.ts` и линтер проходил.

   - `"main"` НЕ указывать (старое CJS-поле). Только `"exports"`:
     ```
     "exports": {
       ".": {
         "types": "./dist/index.d.ts",
     ```

   - `"outDir": "./dist"`, `"rootDir": "./src"`.
   - `"skipLibCheck": true`.

4. **Создать `packages/middleware/tsup.config.ts`** — конфиг сборки:
   - `entry: ['src/index.ts']`.

   - `dts: true` — генерировать `.d.ts` через `tsup`.
   - `sourcemap: true`.
   - `clean: true`.
   - `target: 'node20'`.
   - `splitting: false`.
   - `treeshake: true`.
   - `minify: false` (минификация делает unit-тесты багов сложными; Task 8 проверит bundle size через `--minify` опцию или внешний прогон, см. tech-spec AC "<30 KB minified+gzip").

5. **Создать `.eslintrc.cjs`** в корне (именно `.cjs`, потому что `"type": "module"` сломает CommonJS-конфиг ESLint 8):

   - `extends: ['eslint:recommended', 'plugin:@typescript-eslint/recommended', 'prettier']`.
   - `plugins: ['@typescript-eslint']`.
   - `env: { node: true, es2022: true }`.
   - `ignorePatterns: ['dist/', 'node_modules/', 'cache/', 'artifacts/', 'typechain-types/', 'coverage/', '**/*.cjs']`.
   - Минимальные строгие правила: `'@typescript-eslint/no-unused-vars': ['error', { argsIgnorePattern: '^_' }]`, `'no-console': 'warn'`, `'@typescript-eslint/no-explicit-any': 'warn'`.

6. **Создать `.prettierrc`** в корне:

   - Добавить в корневой `package.json` корневую инструкцию `prepare: "husky"` (huskyv9 — без `husky install`).

Из tech-spec `Verify-smoke`:
```

```
Все три этапа должны вернуть exit code 0. Дополнительно вручную:

- `cat packages/middleware/package.json | node -e "const p=JSON.parse(require('fs').readFileSync(0,'utf8')); if(p.exports['.'].require) {console.error('FAIL: require branch present'); process.exit(1);} if(p.type !== 'module') {console.error('FAIL: not ESM'); process.exit(1);} console.log('ESM-only invariant OK')"` — печатает `ESM-only invariant OK`.

(Прямой smoke pre-commit hook'а без реального staged-файла затруднён; этот surrogate-тест достаточен.)

**Edge cases:**
- npm workspaces ругается warning'ом на отсутствующие `apps/*` и `contracts` workspaces — это **ожидаемо**, не блокирует install. Можно либо игнорировать warning, либо временно использовать `["packages/*"]` и расширить workspaces в Task 2. **Решение**: оставляем сразу `["packages/*", "apps/*", "contracts"]` чтобы избежать миграции workspace-конфига позже; npm 10+ толерантен к ненайденным glob-паттернам.
- gitleaks НЕ устанавливается через npm — должен быть в системе (brew install gitleaks). Если отсутствует, pre-commit упадёт. В `.husky/pre-commit` добавить проверку `command -v gitleaks` с понятным сообщением и инструкцией установки.
- TypeScript `verbatimModuleSyntax: true` требует явных `import type` для типов — это намеренно для ESM-only стиля, но создаст шум в Task 6+ когда появится реальный код. Команды должны быть готовы к этому.
- `tsup` с `dts: true` требует `typescript` как peer — он в корневых devDeps, через workspaces подхватится.
- ESLint 8 → 9 migration не делаем (ESLint 9 flat config — отдельная задача, post-MVP).
- `"exports"` map в ESM-only пакете БЕЗ `"main"`-фолбэка может сломать старые Node.js < 12 — мы требуем `>=20`, эта проблема не актуальна.
- `husky@9` ушёл от `husky install` к `husky` в `prepare`-скрипте — указано выше.

**Implementation hints:**
- Версии devDeps закреплять минорной маской (`^`), мажорные — фиксировать. Зафиксировать lock через `npm install` (создаст `package-lock.json`).
- Если `@typescript-eslint/parser` не подхватывает `project` setting — добавить `tsconfigRootDir: __dirname` в parserOptions.
- Для `tsup` с ESM-only выходом убедиться что НЕ передаётся `--format cjs` ни через конфиг, ни через CLI.
- `husky@9` setup: после `npm install` husky сам инициализируется через `prepare`; нужно вручную создать `.husky/pre-commit` (`husky init` — опциональный helper).
- gitleaks отсутствие — corner case: оформить как warning в hook с `exit 0`, чтобы dev на машине без gitleaks мог коммитить, но CI заблокирует. **Решение**: в hook'е fail-hard (`exit 1` если нет gitleaks). README в Task 11 объяснит установку. **Альтернатива**: fail-soft с предупреждением, чтобы не блокировать локалку, а CI закроет gap. Выбрать **fail-hard** — это безопасный default; для x402 потеря приватного ключа в коммите критична.
- Не плодить лишние devDeps: `lint-staged`, `commitlint`, `release-it` — НЕ в эту задачу.
