# Task 1: Monorepo scaffolding (ESM-only)

## Required Skills

Перед выполнением задачи загрузи:
- `/skill:infrastructure-setup` — [skills/infrastructure-setup/SKILL.md](~/.claude/skills/infrastructure-setup/SKILL.md)

## Description

Поднять с нуля корневой npm workspace монорепо `universal-paywall` и каркас пакета `packages/middleware` как ESM-only TypeScript-проекта, готового под последующие волны (контракты, middleware-модули, тесты, деплой). Сейчас в репозитории нет ни `package.json`, ни `packages/`, ни линтера, ни pre-commit — только `.gitignore`, `CLAUDE.md` и папка `work/`. Эта задача создаёт фундамент, на который опираются все остальные задачи feature x402-agent-payment.

Ключевые архитектурные ограничения (tech-spec, Solution + Acceptance Criteria):
- `@universal-paywall/middleware` публикуется как **ESM-only** (`"type": "module"`, exports map без CJS-ветки), `engines.node: ">=20"`.
- Сборка через **tsup** (D7 косвенно — TypeScript-native стек; tsup явно указан в Dependencies → "New (middleware)").
- TypeScript strict-режим.
- `npm install && npm run lint && npm run build --workspace=packages/middleware` должен проходить.
- `npm test --workspace=@universal-paywall/middleware` должен запускаться **сразу же после T1** (даже если тестов ещё нет — vitest должен быть установлен и просто отрапортовать "no tests found" без падения с "command not found"). Это требование §9 iteration-3-addendum: T6 — первая задача, которая реально пишет vitest-тесты, и она должна работать без дополнительных `npm install` команд.
- gitleaks как pre-commit hook (см. patterns.md "Security Gates → Pre-commit: gitleaks").
- ESLint + Prettier — pre-commit (TypeScript typecheck + ESLint per patterns.md).
- В `packages/middleware/package.json` должны существовать скрипты `"test": "vitest run"` и `"test:e2e": "vitest run src/__tests__/integration/"`, и они должны работать (vitest установлен).

**§9 iteration-3-addendum:** T1 устанавливает **полный devDependency-набор**, который понадобится T6+:
- На корневом уровне (devDependencies в `/package.json`): `typescript`, `eslint`, `@typescript-eslint/*`, `prettier`, `husky`, `tsup`, **`vitest`** (чтобы CI/локально можно было запустить `npx vitest` из корня).
- На уровне `packages/middleware/devDependencies`: `tsup`, `typescript`, `tsx`, `vitest`, `ajv`, `pino`, `winston`. После `npm install` `npm test --workspace=@universal-paywall/middleware` исполняется без ошибок (даже если тестов ещё нет — vitest напечатает "No test files found" и вернёт exit 0 с флагом `--passWithNoTests`, либо exit 1 без флага — оба варианта означают, что бинарь найден).

`viem`, `ajv` (как production dep), реальные тестовые fixtures появятся в T6+. Здесь — инфраструктурный минимум + полный набор test/build/log-dev-зависимостей, чтобы все последующие волны не тратили шаги на `npm install` новых dev-зависимостей.

2. **Создать `packages/middleware/package.json`** с:

   - `"version": "0.0.0"` (бамп до `0.1.0-alpha.0` будет в Task 16).
   - `"type": "module"`.
   - `"main"` НЕ указывать (старое CJS-поле). Только `"exports"`:
     ```
     "exports": {
       ".": {
         "types": "./dist/index.d.ts",
         "import": "./dist/index.js"
       }
     }
     ```
     Никакой `require` ветки — это требование "ESM-only" в acceptance criteria и в названии задачи.
   - `"types": "./dist/index.d.ts"`.
   - `"files": ["dist", "README.md"]`.
   - `"engines": { "node": ">=20" }`.
   - `"sideEffects": false`.
   - Скрипты (обязательно включить ВСЕ перечисленные — все должны быть runnable сразу после `npm install` благодаря devDeps ниже). В `scripts` нужно также добавить sibling-комментарий через npm-tolerated ключ `"//test"`, фиксирующий правило о `--passWithNoTests` (см. §5 T1 iteration-4-addendum):
     ```json
     "scripts": {
       "build": "tsup",
       "dev": "tsup --watch",
       "typecheck": "tsc --noEmit",
       "clean": "rm -rf dist",
       "//test": "keep --passWithNoTests until T6 lands first vitest test; do not remove",
       "test": "vitest run --passWithNoTests",
       "test:e2e": "vitest run src/__tests__/integration/ --passWithNoTests"
     }
     ```
     Флаг `--passWithNoTests` нужен сейчас: в T1 тестов ещё нет, и без флага vitest упадёт с exit 1 "no test files found". Когда T6 запишет первый тест, флаг становится безвредным (vitest проигнорирует его при наличии тестов) — но **до того как T6 закоммитит первый тест, удалять флаг запрещено** (это сломает T1→T5 smoke и сборку CI). Sibling-ключ `"//test"` — это npm-валидный JSON-комментарий (npm игнорирует ключи скриптов, имена которых начинаются с `//`), он остаётся в файле как self-documenting guard. Удаление флага должно происходить в одном коммите с появлением первого vitest-теста в T6 (не раньше).
   - **`devDependencies` (packages/middleware)** — фикс по §9 iteration-3-addendum:
     ```json
     "devDependencies": {
       "tsup": "^8.0.0",
       "typescript": "^5.4.0",
       "tsx": "^4.7.0",
       "vitest": "^1.5.0",
       "ajv": "^8.12.0",
       "pino": "^9.0.0",
       "winston": "^3.11.0"
     }
     ```
     Назначение каждой зависимости:
     - `tsup` — bundler (используется build-скриптом).
     - `typescript` — peer для `tsup --dts` и для `tsc --noEmit` typecheck.
     - `tsx` — ESM-aware runner для скриптов (`scripts/register.ts` в T11, `verify-usdc-eip3009.ts` в T3 через Hardhat — но tsx может пригодиться для локального дебага).
     - `vitest` — test runner (используется T6+; ставится здесь, чтобы T6 не делал второй раунд `npm install`).
     - `ajv` — JSON schema validator (используется T9 для валидации 402 response body против vendored x402 schema; ставится в devDeps т.к. в production-коде middleware ajv не нужен, только в тестах).
     - `pino`, `winston` — logger зависимости для smoke-тестов redaction в T6 (relayer-key.ts должен корректно redact'ить при сериализации через pino/winston; smoke-тест: `JSON.stringify(pinoSerializers.err(opaqueKey))` не содержит секрет). Ставятся как devDeps, т.к. в production middleware не зависит от конкретного logger'а — пользователь передаёт свой SecurityLogger через opts.
   - `peerDependencies` и `dependencies` оставить пустыми (`viem`, runtime `ajv` если потребуется, и др. появятся в Wave 4+).
   - `publishConfig.access: "public"`, `publishConfig.provenance: true` (для Task 16).

   - `"target": "ES2022"`, `"module": "ESNext"`, `"moduleResolution": "Bundler"`.
   - `"strict": true`, плюс явные доп. флаги: `"noUncheckedIndexedAccess": true`, `"noImplicitOverride": true`, `"exactOptionalPropertyTypes": true`.
   - `"esModuleInterop": true`, `"verbatimModuleSyntax": true`.
   - `"declaration": true`, `"declarationMap": true`, `"sourceMap": true`.
   - `"outDir": "./dist"`, `"rootDir": "./src"`.
   - `"skipLibCheck": true`.
   - `"include": ["src"]`, `"exclude": ["node_modules", "dist", "src/__tests__", "**/*.test.ts"]` — путь `src/__tests__` (а не корневой `__tests__`) согласован с canonical test-tree из systemic-fix §2.

4. **Создать `packages/middleware/tsup.config.ts`** — конфиг сборки:
   - `entry: ['src/index.ts']`.
   - `format: ['esm']` (никакого `cjs`).
   - `dts: true` — генерировать `.d.ts` через `tsup`.
   - `sourcemap: true`.
   - `clean: true`.
   - `target: 'node20'`.
   - `splitting: false`.
   - `treeshake: true`.
   - `minify: false` (минификация делает unit-тесты багов сложными; Task 8 проверит bundle size через `--minify` опцию или внешний прогон, см. tech-spec AC "<30 KB minified+gzip").

5. **Создать `.eslintrc.cjs`** в корне (именно `.cjs`, потому что `"type": "module"` сломает CommonJS-конфиг ESLint 8):
   - `parser: '@typescript-eslint/parser'`.
   - `parserOptions: { ecmaVersion: 2022, sourceType: 'module', project: ['./packages/middleware/tsconfig.json'] }` — **только middleware**; contracts/ покрывается hardhat-инструментами.
   - `extends: ['eslint:recommended', 'plugin:@typescript-eslint/recommended', 'prettier']`.
   - `plugins: ['@typescript-eslint']`.
   - `env: { node: true, es2022: true }`.
   - `ignorePatterns: ['dist/', 'node_modules/', 'cache/', 'artifacts/', 'typechain-types/', 'coverage/', 'contracts/', 'apps/', '**/*.cjs']` — игнорируем `contracts/` и `apps/` явно, чтобы ESLint не пытался парсить Solidity-пути и пустые скаффолды.
   - Минимальные строгие правила: `'@typescript-eslint/no-unused-vars': ['error', { argsIgnorePattern: '^_' }]`, `'no-console': 'warn'`, `'@typescript-eslint/no-explicit-any': 'warn'`.

6. **Создать `.prettierrc`** в корне:
   - `singleQuote: true`, `semi: true`, `trailingComma: 'all'`, `printWidth: 100`, `tabWidth: 2`, `arrowParens: 'always'`.
   - Создать также `.prettierignore` со списком `dist/`, `node_modules/`, `cache/`, `artifacts/`, `coverage/`, `typechain-types/`, `*.sol` (sol форматируется отдельно в Task 2).

7. **Создать `.husky/pre-commit`** с двумя проверками. Lint в hook'е запускается **только для middleware** — не для всего репозитория и не для contracts/. Псевдокод:
   ```sh
   #!/usr/bin/env sh
   # gitleaks must be installed system-wide (brew install gitleaks) — see deployment.md
   command -v gitleaks >/dev/null 2>&1 || { echo "gitleaks not found — install via 'brew install gitleaks'"; exit 1; }
   gitleaks protect --staged --redact || exit 1
   # Lint only middleware (contracts has its own hardhat lint, run separately)
   npm run lint --silent || exit 1
   ```
   - `npm run lint` в корне уже scoped на `packages/middleware` (см. п.1), поэтому hook не трогает `contracts/`.
   - Сделать файл исполняемым (`chmod +x .husky/pre-commit`).
   - Корневой `package.json` содержит `prepare: "husky"` (husky v9 — без отдельного `husky install`).

8. **Создать `.gitleaks.toml`** (минимальный конфиг) в корне ИЛИ полагаться на default gitleaks rules. Решение: явно создать `.gitleaks.toml` с `[extend] useDefault = true` плюс пустой `[allowlist]` — это явный артефакт, который будущие задачи смогут расширять (например, добавлять регекспы для arc-testnet RPC URL в allowlist если они появятся в README).

9. **Smoke-проверка**: после установки выполнить
   ```
   npm install
   npm run lint
   npm run build --workspace=packages/middleware
   npm test --workspace=@universal-paywall/middleware
   ```
   Все четыре команды должны завершаться кодом 0. `dist/index.js` и `dist/index.d.ts` должны существовать после build. `npm test` пройдёт благодаря `--passWithNoTests` (vitest установлен, тестов ещё нет — exit 0).

10. **Validation guard**: убедиться, что в `packages/middleware/package.json` НЕТ полей `"main"`, `"module"` без exports-маппинга, и НЕТ ветки `"require"` в exports — это автоматический ESM-only invariant (часть AC tech-spec). Также убедиться что в `scripts` есть оба ключа `test` и `test:e2e`, и что `devDependencies` содержит `vitest`, `tsup`, `typescript`, `tsx`, `ajv`, `pino`, `winston` (§9 iteration-3-addendum).
