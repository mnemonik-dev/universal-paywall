# Task 10: Forked integration + Arc Testnet e2e (gated)

## Description

Build two end-to-end test suites that exercise the full x402 payment pipeline (middleware + factory + vault + USDC) across both supported adapters and both runtime environments:

1. **`contracts/test/integration/forked-e2e.test.ts`** — runs in CI by default (no env gate). Inside one Hardhat node process, it deploys the mock USDC (with EIP-3009), `PaymentSplitterFactory`, and a developer vault (clone). It then spins up TWO in-process HTTP servers wired through the middleware: one using the Node `http` adapter (`withPaywall`) and one using the Fastify adapter (`fastifyPaywall`). The test exercises the happy path through BOTH adapters and runs a CROSS-ADAPTER replay test that pays via Node http, then retries the IDENTICAL `X-PAYMENT` header on the Fastify endpoint in the SAME process — asserting `402 nonce_already_used` to prove the process-singleton `NonceStore` claim from D5. It also exercises rejection branches (`vault_not_deployed`, `paused`) and asserts that on success: the response is 200, the vault USDC balance increased by `value`, and on-chain `usdc.authorizationState(from, nonce) === true`.

2. **`packages/middleware/__tests__/integration/arc-testnet-e2e.test.ts`** — gated by `ARC_TESTNET_E2E=1` (runs in the nightly job, skipped otherwise). Uses a real EIP-3009 signer against the factory + vault deployed on live Arc Testnet via the public RPC. Asserts 402 body conforms to the vendored x402 v1 JSON Schema (via ajv), that the 200 response carries an `X-PAYMENT-RESPONSE` header, and that the on-chain vault USDC balance increased after settle.

Together these two suites give CI-default coverage of the wired-up system on a forked node and nightly coverage of the production-like environment. They directly satisfy two ACs in tech-spec Acceptance Criteria: "Forked integration test passes in CI without env flag" and "Live Arc Testnet test passes when `ARC_TESTNET_E2E=1`".

## What to do

1. **Add an integration test directory under `packages/middleware/`** (`__tests__/integration/`) and add `contracts/test/integration/` if not present. Wire the vitest config and hardhat config so each suite is discovered by the respective runner.

2. **Build the forked-e2e test harness:**
   - Start a Hardhat in-process network (default `hardhat` network) within the test.
   - Deploy: `MockUsdcEip3009` (test mock from Task 4), `PaymentVaultImpl` (impl), `PaymentSplitterFactory` (with `usdc`, `vaultImpl`, `platformTreasury`, default `feeBps = 50`).
   - Call `factory.register()` from a "developer" signer to deploy that developer's vault clone.
   - Mint mock USDC to a "payer" signer; also pre-fund the relayer EOA with mock USDC so settle has gas (mock USDC on the local node has no gas semantics, so any non-zero balance suffices for the relayer balance pre-check).
   - Build the middleware `PaywallConfig` pointing at the local Hardhat RPC (`http://127.0.0.1:<port>` or the in-process provider URL exposed by Hardhat), wrapping the relayer key in `OpaqueRelayerKey`, and using a `network` config whose `factoryAddress`, `vaultImplAddress`, `usdcAddress`, `chainId`, `usdcEip712Name`, `usdcEip712Version` correspond to what was deployed.
   - Spin up a Node `http.createServer(withPaywall(handler, config))` on an ephemeral port AND a Fastify app with `fastify.register(fastifyPaywall(config))` on a second ephemeral port — both in the same process, both sharing the same middleware module instance (so they share the process-singleton `NonceStore` per D5).

3. **Write the test cases in the forked suite:**
   - `node-http happy path`: payer signs EIP-3009 authorization (`from = payer`, `to = developerVault`, `value = maxAmountRequired`, fresh `nonce`, valid time window); first GET to Node http server without `X-PAYMENT` returns 402 with the proper body (ajv schema check); second GET with the encoded `X-PAYMENT` returns 200 with an `X-PAYMENT-RESPONSE` header; assert vault USDC balance grew by exactly `value`; assert `usdc.authorizationState(from, nonce) === true`.
   - `fastify happy path`: same flow against the Fastify endpoint with a fresh nonce.
   - `cross-adapter replay`: pay successfully via the Node http server, then send the EXACT SAME `X-PAYMENT` header to the Fastify endpoint — assert 402 response with body `error: "nonce_already_used"`. This is the key invariant of D5 / Task 8's process-singleton `NonceStore`.
   - `vault_not_deployed branch`: use a developer EOA that never called `factory.register()`; configure the middleware against this developer; the 402 settle attempt rejects with body `error: "vault_not_deployed"`.
   - `paused branch`: call `factory.pause()` from owner; after the 5-second cache TTL (or call a test-only cache invalidator if one exists on the core), the next signed request rejects with body `error: "paused"`.

4. **Build the live Arc Testnet e2e test (`arc-testnet-e2e.test.ts`):**
   - Gate the entire describe block on `process.env.ARC_TESTNET_E2E === '1'` (vitest `describe.skipIf`/`describe.runIf` pattern). Skipped suite must not fail CI when the env var is absent.
   - Require env vars: `ARC_TESTNET_RPC_URL`, `ARC_TESTNET_PAYER_PK`, `ARC_TESTNET_RELAYER_PK`, `ARC_TESTNET_DEVELOPER_EOA`, `ARC_TESTNET_FACTORY_ADDRESS` (the addresses come from Task 11's deploy). Fail with a clear error if any required env var is missing while the flag is set.
   - Build a `PaywallConfig` targeting Arc Testnet (real `NETWORKS['eip155:5042002']`).
   - Spin up a single Node http server wrapping `withPaywall(handler, config)`.
   - Sign a real EIP-3009 authorization with the payer key (use the real Arc USDC `chainId`, `verifyingContract`, `name`, `version` from `NETWORKS`).
   - GET without header → 402; ajv-validate the body against the vendored x402 v1 schema fixture from Task 9 (`packages/middleware/__tests__/fixtures/x402-v1.schema.json`).
   - GET with the signed header → 200, assert `X-PAYMENT-RESPONSE` header is present and base64-JSON-decodable to `{success: true, transaction: 0x..., network, payer}`.
   - Read `usdc.balanceOf(vault)` before and after; assert it increased by `value`.

5. **Wire the test commands:**
   - Forked suite runs under the existing `cd contracts && npx hardhat test`. Confirm `forked-e2e.test.ts` is picked up by the default Hardhat test pattern.
   - Arc Testnet suite runs under `npm test --workspace=packages/middleware` only when `ARC_TESTNET_E2E=1`. Add an `npm run test:e2e --workspace=packages/middleware` script entry if not already present (Task 1 may have stubbed it).

## Details

**Files (current state and what to change):**

- `contracts/test/integration/forked-e2e.test.ts` — NEW. Hardhat + chai (matches the Task 5 contract-test style). Inside one test process: deploys `MockUsdcEip3009`, `PaymentVaultImpl`, `PaymentSplitterFactory`; registers a developer; starts a Node `http.createServer` on `0.0.0.0:0` AND a Fastify app on `0.0.0.0:0` (both in the same Node process so the middleware module's process-singleton `NonceStore` is genuinely shared); each adapter wraps the same `handler = (req, res) => res.end('ok')`. Cases: node-http happy, fastify happy, cross-adapter replay, vault_not_deployed, paused.

- `packages/middleware/__tests__/integration/arc-testnet-e2e.test.ts` — NEW. Vitest. `describe.runIf(process.env.ARC_TESTNET_E2E === '1', ...)` (or equivalent skip-when-unset pattern). Reads env: `ARC_TESTNET_RPC_URL`, `ARC_TESTNET_PAYER_PK`, `ARC_TESTNET_RELAYER_PK`, `ARC_TESTNET_DEVELOPER_EOA`, `ARC_TESTNET_FACTORY_ADDRESS`. Constructs `PaywallConfig` for the real network. Signs a real EIP-3009 authorization with viem. Uses ajv to validate the 402 body against `__tests__/fixtures/x402-v1.schema.json` (vendored in Task 9). Asserts 200 + `X-PAYMENT-RESPONSE` header + on-chain vault USDC balance delta.

**Dependencies:**
- Task 5 (contract tests) provides the contracts deployment patterns and test fixtures that the forked suite reuses.
- Task 8 (core + adapters + index) is the surface under test — both `withPaywall` and `fastifyPaywall` must be exported from `@universal-paywall/middleware`.
- Task 9 (vitest config + ajv + vendored x402 v1 schema fixture) — schema fixture is consumed by the Arc Testnet suite.
- Task 11 (deploy + register CLI) — provides the live Arc Testnet factory/vault addresses the gated e2e suite hits. The gated suite is allowed to land before Task 11 is run because it is skipped by default — but it will only pass post-deploy.
- `viem` 2.x for EIP-712 signing (payer key) and RPC reads.
- `ajv` for schema validation in the Arc Testnet suite.
- `fastify` and Node `http` for the adapter harnesses; both already required by `packages/middleware`.

**Edge cases:**
- Hardhat in-process node port allocation: bind to `:0` and read the actual port; do not hardcode 8545 (CI may have something on it).
- The mock USDC may not emit `name()` / `version()` matching the production USDC; the test must construct EIP-712 domain values from what the mock advertises (read from the contract or wire constants).
- The middleware's factory-state cache has a 5 s TTL (D12). The `paused` branch test must either wait > 5 s after `factory.pause()` or call a test-only invalidator on the core if exposed; if none is exposed, do the simple thing — wait. Document the wait so the suite isn't slow by surprise.
- The cross-adapter replay must use the EXACT bytes of `X-PAYMENT` (do not re-encode after the first request — re-encoding can produce different base64 padding, breaking the test's intent).
- The Arc Testnet suite must NOT spend more USDC than the payer holds; choose a tiny `value` (e.g., `10_000` micro-USDC = $0.01).
- The Arc Testnet suite must close the server and not leave open handles (otherwise vitest hangs).

**Implementation hints:**
- Sign EIP-3009 with viem `walletClient.signTypedData({domain, types, primaryType: 'TransferWithAuthorization', message})`. Types from EIP-3009; primary type name is `TransferWithAuthorization`.
- For nonce, use a random 32-byte hex per test (`bytes32(uuid)` works; `viem.toHex(crypto.getRandomValues(new Uint8Array(32)))`).
- `validAfter = 0`, `validBefore = Math.floor(Date.now()/1000) + 600` (10 min margin so the 5 s safety check at verify time never trips).
- For the cross-adapter test: capture the raw header string after the first 200 response, then issue a fresh GET to the OTHER adapter with the captured string as `X-PAYMENT`. Do not re-serialize.
- For the `paused` branch, the cleanest approach: use vitest/chai's `await new Promise(r => setTimeout(r, 5500))` after `factory.pause()` so the middleware cache expires. Mark the case with `slow` if vitest supports per-case timeout.
- Skip-without-flag pattern in vitest: `describe.runIf(condition, ...)` is the supported idiom (v0.34+). If unavailable, use `describe.skipIf(condition === false, ...)` or wrap `it` calls in `(condition ? it : it.skip)`.
