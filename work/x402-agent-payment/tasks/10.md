# Task 10: Forked integration + Arc Testnet e2e (gated)

## Description

Build two end-to-end test suites that exercise the full x402 payment pipeline (middleware + factory + vault + USDC) across both supported adapters and both runtime environments:

1. **`contracts/test/integration/forked-e2e.test.ts`** — runs in CI by default (no env gate). Inside one Hardhat node process, it deploys the mock USDC (with EIP-3009), `PaymentSplitterFactory`, and a developer vault (clone). It then spins up TWO in-process HTTP servers wired through the **built** middleware (`packages/middleware/dist/index.js` — Hardhat runs CJS by default, so this test imports the compiled bundle, not source): one using the Node `http` adapter (`withPaywall`) and one using the Fastify adapter (`fastifyPaywall`). The test exercises the happy path through BOTH adapters and runs a CROSS-ADAPTER replay test that pays via Node http, then retries the IDENTICAL `X-PAYMENT` header on the Fastify endpoint in the SAME process — asserting `402 nonce_already_used` to prove the process-singleton `NonceStore` claim from D5. It also exercises rejection branches (`vault_not_deployed`, `paused`) and asserts that on success: the response is 200, the vault USDC balance increased by `value`, and on-chain `usdc.authorizationState(from, nonce) === true`.

2. **`packages/middleware/src/__tests__/integration/arc-testnet-e2e.test.ts`** — gated by `ARC_TESTNET_E2E=1` (runs in the nightly job, skipped otherwise). Uses a real EIP-3009 signer against the factory + vault deployed on live Arc Testnet via the public RPC. Asserts 402 body conforms to the vendored x402 v1 JSON Schema (via ajv), that the 200 response carries an `X-PAYMENT-RESPONSE` header, and that the on-chain vault USDC balance increased after settle.

Together these two suites give CI-default coverage of the wired-up system on a forked node and nightly coverage of the production-like environment. They directly satisfy two ACs in tech-spec Acceptance Criteria: "Forked integration test passes in CI without env flag" and "Live Arc Testnet test passes when `ARC_TESTNET_E2E=1`".

## What to do

1. **Place integration test files in the canonical trees:**
   - Vitest e2e (Arc Testnet) lives under `packages/middleware/src/__tests__/integration/arc-testnet-e2e.test.ts` (per systemic-fix §2 — single `src/__tests__/` tree). The `test:e2e` npm script that resolves this path was already added to `packages/middleware/package.json` by Task 1 (per systemic-fix §9) — this task does NOT modify package.json.
   - Forked-e2e (Hardhat + mocha) lives under `contracts/test/integration/forked-e2e.test.ts`. Confirm Hardhat's default test glob picks it up.

2. **Build the forked-e2e test harness:**
   - Start a Hardhat in-process network (default `hardhat` network) within the test.
   - Deploy: `MockUsdcEip3009` (test mock from Task 4), `PaymentVaultImpl` (impl), `PaymentSplitterFactory` (with `usdc`, `vaultImpl`, `platformTreasury`, default `feeBps = 50`).
   - Call `factory.register()` from a "developer" signer to deploy that developer's vault clone.
   - Mint mock USDC to a "payer" signer; also pre-fund the relayer EOA with mock USDC so settle has gas (mock USDC on the local node has no gas semantics, so any non-zero balance suffices for the relayer balance pre-check).
   - **Import the middleware from the BUILT bundle**, not source: `require('../../../packages/middleware/dist/index.js')` (Hardhat runs CJS by default and source is ESM-targeted). If a direct CJS `require` errors, fall back to dynamic `await import('@universal-paywall/middleware')` inside the test body — the workspace alias resolves to the built `dist/` per the package's `exports` field.
   - Build the middleware `PaywallConfig` pointing at the local Hardhat RPC (`http://127.0.0.1:<port>` or the in-process provider URL exposed by Hardhat), wrapping the relayer key in `OpaqueRelayerKey`, and using a `network` config whose `factoryAddress`, `vaultImplAddress`, `usdcAddress`, `chainId`, `usdcEip712Name`, `usdcEip712Version` correspond to what was deployed. Because `NETWORKS['arc-testnet']` still holds `0x0` sentinel placeholders pre-deploy (per systemic-fix §13), the forked test must override `factoryAddress`/`vaultImplAddress`/`usdcAddress` via test-local config — NOT rely on the registry.
   - Spin up a Node `http.createServer(withPaywall(handler, config))` on an ephemeral port AND a Fastify app with `fastify.register(fastifyPaywall(config))` on a second ephemeral port — both in the same process, both sharing the same middleware module instance (so they share the process-singleton `NonceStore` per D5).
   - **Mocha timeout:** set `this.timeout(60000)` at the top of the describe suite. The `paused` branch requires a ~5.5 s wait for the factory-state cache TTL to expire, which exceeds Mocha's default 2 s timeout.

3. **Write the test cases in the forked suite:**
   - `node-http happy path`: payer signs EIP-3009 authorization (`from = payer`, `to = developerVault`, `value = maxAmountRequired`, fresh `nonce`, valid time window); first GET to Node http server without `X-PAYMENT` returns 402 with the proper body (ajv schema check); second GET with the encoded `X-PAYMENT` returns 200 with an `X-PAYMENT-RESPONSE` header; assert vault USDC balance grew by exactly `value`; assert `usdc.authorizationState(from, nonce) === true`.
   - `fastify happy path`: same flow against the Fastify endpoint with a fresh nonce.
   - `cross-adapter replay`: pay successfully via the Node http server, then send the EXACT SAME `X-PAYMENT` header to the Fastify endpoint — assert 402 response with body `error: "nonce_already_used"` (canonical error string per systemic-fix §4). This is the key invariant of D5 / Task 8's process-singleton `NonceStore`.
   - `vault_not_deployed branch`: use a developer EOA that never called `factory.register()`; configure the middleware against this developer; the 402 settle attempt rejects with body `error: "vault_not_deployed"`.
   - `paused branch`: call `factory.pause()` from owner; wait for the factory-state cache TTL to expire (~5 s) so the next read picks up the paused state, then the next signed request rejects with body `error: "paused"`.

4. **Build the live Arc Testnet e2e test (`arc-testnet-e2e.test.ts`):**
   - Gate the entire describe block on `process.env.ARC_TESTNET_E2E === '1'` (vitest `describe.skipIf`/`describe.runIf` pattern). Skipped suite must not fail CI when the env var is absent.
   - Require env vars (canonical names per systemic-fix §3): `ARC_RPC_URL` (the Arc Testnet RPC, defaults to `https://rpc.testnet.arc.network`), `PAYWALL_RELAYER_KEY` (facilitator relayer signer hex), `ARC_TESTNET_PAYER_PK` (payer signer hex), `ARC_TESTNET_DEVELOPER_EOA` (developer EOA whose vault receives settle), `PAYMENT_SPLITTER_FACTORY_ADDRESS` (live factory address — populated into `NETWORKS` by Task 11's deploy patch). Fail with a clear error if any required env var is missing while the flag is set.
   - Build a `PaywallConfig` targeting Arc Testnet. After Task 11's deploy patch runs, the `NETWORKS['arc-testnet']` registry has the live `factoryAddress` written in place of the sentinel `0x0`; this test reads from `NETWORKS` directly. As a defensive fallback, allow `PAYMENT_SPLITTER_FACTORY_ADDRESS` env override.
   - Wrap the relayer key from `PAYWALL_RELAYER_KEY` in `OpaqueRelayerKey` before placing it in `PaywallConfig.facilitator.relayerKey`.
   - Spin up a single Node http server wrapping `withPaywall(handler, config)`.
   - Sign a real EIP-3009 authorization with the payer key (use the real Arc USDC `chainId`, `verifyingContract`, `name`, `version` from `NETWORKS`).
   - GET without header → 402; ajv-validate the body against the vendored x402 v1 schema fixture from Task 9 at `packages/middleware/src/__tests__/fixtures/x402-v1.schema.json` (canonical path per systemic-fix §2).
   - GET with the signed header → 200, assert `X-PAYMENT-RESPONSE` header is present and base64-JSON-decodable to `{success: true, transaction: 0x..., network, payer}`.
   - Read `usdc.balanceOf(vault)` before and after; assert it increased by `value`.

5. **Wire the test commands:**
   - Forked suite runs under `cd contracts && npx hardhat test`. Confirm `forked-e2e.test.ts` is picked up by the default Hardhat test pattern.
   - Arc Testnet suite is invoked via `ARC_TESTNET_E2E=1 npm run test:e2e --workspace=@universal-paywall/middleware` (script defined by Task 1 — resolves `vitest run src/__tests__/integration/`).

**Edge cases:**
- Hardhat in-process node port allocation: bind to `:0` and read the actual port; do not hardcode 8545 (CI may have something on it).
- ESM/CJS interop: Hardhat runs CJS by default. The forked test imports the built middleware bundle (`packages/middleware/dist/index.js`); if a direct CJS `require` errors due to a module-format mismatch, fall back to dynamic `await import('@universal-paywall/middleware')` inside the test body — the workspace package's `exports` map resolves to the same compiled `dist/` entry. Never import middleware source `.ts` files from Hardhat.
- The mock USDC may not emit `name()` / `version()` matching the production USDC; the test must construct EIP-712 domain values from what the mock advertises (read from the contract or wire constants).
- The middleware's factory-state cache has a 5 s TTL (D12). The `paused` branch test must wait for the cache TTL to elapse before issuing the next request — the canonical pattern is "wait for cache TTL" (the suite's `this.timeout(60000)` accommodates this).
- The cross-adapter replay must use the EXACT bytes of `X-PAYMENT` (do not re-encode after the first request — re-encoding can produce different base64 padding, breaking the test's intent).
- The Arc Testnet suite must NOT spend more USDC than the payer holds; choose a tiny `value` (e.g., `10_000` micro-USDC = $0.01).
- The Arc Testnet suite must close the server and not leave open handles (otherwise vitest hangs).
- `NETWORKS['arc-testnet']` pre-deploy holds `0x0` sentinel addresses. The forked test does NOT consume this registry (it overrides addresses with locally-deployed contracts). The Arc Testnet test relies on Task 11 having patched the sentinels OR on a `PAYMENT_SPLITTER_FACTORY_ADDRESS` env override.

**Implementation hints:**
- Sign EIP-3009 with viem `walletClient.signTypedData({domain, types, primaryType: 'TransferWithAuthorization', message})`. Types from EIP-3009; primary type name is `TransferWithAuthorization`.
- For nonce, use a random 32-byte hex per test (`viem.toHex(crypto.getRandomValues(new Uint8Array(32)))`).
- `validAfter = 0`, `validBefore = Math.floor(Date.now()/1000) + 600` (10 min margin so the 5 s safety check at verify time never trips).
- For the cross-adapter test: capture the raw header string after the first 200 response, then issue a fresh GET to the OTHER adapter with the captured string as `X-PAYMENT`. Do not re-serialize.
- For the `paused` branch: invoke `factory.pause()` and then wait for the middleware's factory-state cache TTL (5 s) to expire before issuing the next signed request. The suite-level `this.timeout(60000)` covers this wait.
- Skip-without-flag pattern in vitest: `describe.runIf(condition, ...)` is the supported idiom (v0.34+). If unavailable, use `describe.skipIf(condition === false, ...)` or wrap `it` calls in `(condition ? it : it.skip)`.
- Canonical 402 error strings (must match verify.ts and tests per systemic-fix §4): `payment_required`, `invalid_signature`, `insufficient_amount`, `authorization_expired`, `authorization_not_yet_valid`, `nonce_already_used`, `network_mismatch`, `paused`, `vault_not_deployed`, `to_mismatch`, `settlement_failed`. This suite asserts `nonce_already_used`, `vault_not_deployed`, `paused` exactly.
