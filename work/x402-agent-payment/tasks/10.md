# Task 10: Forked integration + Arc Testnet e2e (gated)

## Required Skills

Перед выполнением задачи загрузи:

## Description

Build two end-to-end test suites that exercise the full x402 payment pipeline (middleware + factory + vault + USDC) across both supported adapters and both runtime environments. **Both suites run under vitest in the middleware package** — there is NO Hardhat/Mocha test runner involved. The contract toolchain is Foundry (see iteration-4-addendum §1) — `anvil` is the local node and compiled artifacts come from `contracts/out/`.

1. **`packages/middleware/src/__tests__/integration/forked-e2e.test.ts`** — runs in CI by default (no env gate). The test SPAWNS a real `anvil --chain-id 31337 --port $TEST_PORT` as a child process in `beforeAll`, polls TCP readiness on that port, then deploys the mock USDC (with EIP-3009), `PaymentVaultImpl`, and `PaymentSplitterFactory` programmatically via viem's `walletClient.deployContract` — reading the compiled bytecode from `contracts/out/<Contract>.sol/<Contract>.json` (Foundry's `forge build` output). NO Hardhat is involved at any point in this test. It then spins up TWO in-process HTTP servers wired through the middleware (imported via the workspace package `@universal-paywall/middleware`, which resolves through the package's `exports` field): one using the Node `http` adapter (`withPaywall`) and one using the Fastify adapter (`fastifyPaywall`). The middleware's viem `PublicClient` connects to `http://127.0.0.1:<TEST_PORT>` — the same RPC URL the test uses to deploy contracts, so all parties share one source of state. The test exercises the happy path through BOTH adapters and runs a CROSS-ADAPTER replay test that pays via Node http, then retries the IDENTICAL `X-PAYMENT` header on the Fastify endpoint in the SAME process — asserting `402 nonce_already_used` to prove the process-singleton `NonceStore` claim from D5. It also exercises rejection branches (`vault_not_deployed`, `paused`) and asserts that on success: the response is 200, the vault USDC balance increased by `value`, and on-chain `usdc.authorizationState(from, nonce) === true`. `afterAll` kills the spawned `anvil` child process so vitest and CI exit cleanly.

2. **`packages/middleware/src/__tests__/integration/arc-testnet-e2e.test.ts`** — gated by `ARC_TESTNET_E2E=1` (runs in the nightly job, skipped otherwise). Uses a real EIP-3009 signer against the factory + vault deployed on live Arc Testnet via the public RPC. Asserts 402 body conforms to the vendored x402 v1 JSON Schema (via ajv), that the 200 response carries an `X-PAYMENT-RESPONSE` header, and that the on-chain vault USDC balance increased after settle.

Together these two suites give CI-default coverage of the wired-up system on a local anvil node and nightly coverage of the production-like environment. They directly satisfy two ACs in tech-spec Acceptance Criteria: "Forked integration test passes in CI without env flag" and "Live Arc Testnet test passes when `ARC_TESTNET_E2E=1`".

## What to do

1. **Place integration test files in the canonical tree:**
   - Both files live under `packages/middleware/src/__tests__/integration/` — `forked-e2e.test.ts` and `arc-testnet-e2e.test.ts`. There is a SINGLE `src/__tests__/` test tree (per systemic-fix §2) and a SINGLE test runner (vitest).
   - The `test:e2e` npm script that resolves `src/__tests__/integration/` was already added to `packages/middleware/package.json` by Task 1 (per systemic-fix §9) — this task does NOT modify package.json.

2. **Build the forked-e2e test harness:**
   - **Spawn a real anvil child process in `beforeAll`** — do NOT rely on the in-process anvil-via-viem mock. Concretely:
     ```ts
     import { spawn, type ChildProcess } from 'node:child_process';
     import { beforeAll, afterAll } from 'vitest';
     const TEST_PORT = Number(process.env.TEST_PORT ?? 8545);
     let anvilNode: ChildProcess;
     beforeAll(async () => {
       anvilNode = spawn(
         'anvil',
         ['--chain-id', '31337', '--port', String(TEST_PORT)],
         { stdio: ['ignore', 'pipe', 'pipe'] },
       );
       await waitForPort(TEST_PORT, { timeout: 30_000 }); // poll readiness; helper at top of file
     });
     afterAll(() => {
       if (anvilNode && !anvilNode.killed) anvilNode.kill('SIGTERM');
     });
     ```
     `waitForPort` is a small helper inside the test file that opens a TCP socket to `127.0.0.1:<port>` every ~250 ms until it connects or the timeout elapses. Reject with a clear error if anvil never comes up. Timeouts come from `vitest.config.ts` (`testTimeout: 60_000`, `hookTimeout: 30_000`) — do NOT use Mocha's `function()`/`this.timeout(...)` syntax; vitest is the runner.
   - The `TEST_PORT` env var (default `8545`) lets CI parallel-shard pick non-colliding ports. Document this in test setup comments.
   - **Deploy contracts programmatically via viem:** read the compiled `bytecode.object` and `abi` from Foundry's build output:
     - `contracts/out/MockUsdcEip3009.sol/MockUsdcEip3009.json`
     - `contracts/out/PaymentVaultImpl.sol/PaymentVaultImpl.json`
     - `contracts/out/PaymentSplitterFactory.sol/PaymentSplitterFactory.json`
     Use `JSON.parse(fs.readFileSync(...))` and call `walletClient.deployContract({ abi, bytecode, args })` against `http://127.0.0.1:${TEST_PORT}` (viem `WalletClient` + a local anvil-funded EOA). NO Hardhat, NO `forge script` invocation — purely viem against the running anvil. Wait for deployment receipts via `publicClient.waitForTransactionReceipt`.
   - Deploy order: `MockUsdcEip3009` (Task 5 mock), `PaymentVaultImpl`, `PaymentSplitterFactory` (canonical 3-arg constructor per iter-3 addendum §1: `usdc`, `platformTreasury`, `initialFeeBps = 50`; `vaultImpl` is deployed inside the factory constructor, NOT passed in).
   - Call `factory.register()` from a "developer" signer to deploy that developer's vault clone (the factory deploys a minimal proxy via `Clones.cloneDeterministic`).
   - Mint mock USDC to a "payer" signer; also pre-fund the relayer EOA with mock USDC so settle has gas (mock USDC on the local node has no native-gas semantics, so any non-zero balance suffices for the relayer balance pre-check — the iter-3 addendum §7 threshold is `1_000_000n` = 1 USDC).
   - **Import the middleware from the workspace package**, NOT from a relative `dist/` path: `import { withPaywall, fastifyPaywall } from '@universal-paywall/middleware';`. The workspace `exports` field resolves to the built bundle. Vitest in the middleware package runs ESM natively, so no CJS interop is needed.
   - Build the middleware `PaywallConfig` pointing at `http://127.0.0.1:<TEST_PORT>` (the spawned anvil), wrapping the relayer key in `OpaqueRelayerKey`, and using a `network` config whose `factoryAddress`, `vaultImplAddress`, `usdcAddress`, `chainId`, `usdcEip712Name`, `usdcEip712Version` correspond to what was deployed. The middleware's internal viem `PublicClient` MUST be constructed with `http('http://127.0.0.1:' + TEST_PORT)` so middleware and test agree on chain state. Because `NETWORKS['arc-testnet']` still holds `0x0` sentinel placeholders pre-deploy (per systemic-fix §13), the forked test must override `factoryAddress`/`vaultImplAddress`/`usdcAddress` via test-local config — NOT rely on the registry.
   - Spin up a Node `http.createServer(withPaywall(handler, config))` on an ephemeral port AND a Fastify app with `fastify.register(fastifyPaywall(config))` on a second ephemeral port — both in the same process, both sharing the same middleware module instance (so they share the process-singleton `NonceStore` per D5).
   - **Timeouts are configured via `vitest.config.ts`** (Task 9 owns the config). The `paused` branch requires a ~5.5 s wait for the factory-state cache TTL to expire; `testTimeout: 60_000` accommodates this. `beforeAll` spawn-and-ready can take 5-15 s on cold CI machines; `hookTimeout: 30_000` accommodates this. Do NOT inline timeouts via Mocha's `function() { this.timeout(...) }` — that's Mocha syntax and will be ignored under vitest.

3. **Write the test cases in the forked suite:**
   - `node-http happy path`: payer signs EIP-3009 authorization (`from = payer`, `to = developerVault`, `value = maxAmountRequired`, fresh `nonce`, valid time window); first GET to Node http server without `X-PAYMENT` returns 402 with the proper body (ajv schema check); second GET with the encoded `X-PAYMENT` returns 200 with an `X-PAYMENT-RESPONSE` header; assert vault USDC balance grew by exactly `value`; assert `usdc.authorizationState(from, nonce) === true`.
   - `fastify happy path`: same flow against the Fastify endpoint with a fresh nonce.
   - `cross-adapter replay`: pay successfully via the Node http server, then send the EXACT SAME `X-PAYMENT` header to the Fastify endpoint — assert 402 response with body `error: "nonce_already_used"` (canonical error string per systemic-fix §4). This is the key invariant of D5 / Task 8's process-singleton `NonceStore`.
   - `vault_not_deployed branch`: use a developer EOA that never called `factory.register()`; configure the middleware against this developer; the 402 settle attempt rejects with body `error: "vault_not_deployed"`.
   - `paused branch`: call `factory.pause()` from owner; wait for the factory-state cache TTL to expire (~5 s) so the next read picks up the paused state, then the next signed request rejects with body `error: "paused"`.

4. **Build the live Arc Testnet e2e test (`arc-testnet-e2e.test.ts`):**
   - Gate the entire describe block on `process.env.ARC_TESTNET_E2E === '1'`. Use vitest's `describe.skipIf(process.env.ARC_TESTNET_E2E !== '1', 'arc testnet e2e', () => {...})` — i.e. skip-when-NOT-set so the suite is silently skipped under the default CI environment. If `describe.skipIf` is unavailable in the installed vitest version, the equivalent pattern is: at the top of the file, `const SHOULD_RUN = process.env.ARC_TESTNET_E2E === '1';` and then `(SHOULD_RUN ? describe : describe.skip)('arc testnet e2e', () => {...})`. Either way, the assertion in the TDD anchor is: with `ARC_TESTNET_E2E` unset (or any value other than the string `'1'`), the suite is reported as skipped — NOT as failed, NOT as zero tests run.
   - Require env vars (canonical names per systemic-fix §3): `ARC_RPC_URL` (the Arc Testnet RPC, defaults to `https://rpc.testnet.arc.network`), `PAYWALL_RELAYER_KEY` (facilitator relayer signer hex), `ARC_TESTNET_PAYER_PK` (payer signer hex), `ARC_TESTNET_DEVELOPER_EOA` (developer EOA whose vault receives settle), `PAYMENT_SPLITTER_FACTORY_ADDRESS` (live factory address — populated into `NETWORKS` by Task 11's deploy patch). Fail with a clear error if any required env var is missing while the flag is set.
   - Build a `PaywallConfig` targeting Arc Testnet. After Task 11's deploy patch runs, the `NETWORKS['arc-testnet']` registry has the live `factoryAddress` written in place of the sentinel `0x0`; this test reads from `NETWORKS` directly. As a defensive fallback, allow `PAYMENT_SPLITTER_FACTORY_ADDRESS` env override.
   - Wrap the relayer key from `PAYWALL_RELAYER_KEY` in `OpaqueRelayerKey` before placing it in `PaywallConfig.facilitator.relayerKey`.
   - Spin up a single Node http server wrapping `withPaywall(handler, config)`.
   - Sign a real EIP-3009 authorization with the payer key (use the real Arc USDC `chainId`, `verifyingContract`, `name`, `version` from `NETWORKS`).
   - GET without header → 402; ajv-validate the body against the vendored x402 v1 schema fixture from Task 9 at `packages/middleware/src/__tests__/fixtures/x402-v1.schema.json` (canonical path per systemic-fix §2).
   - GET with the signed header → 200, assert `X-PAYMENT-RESPONSE` header is present and base64-JSON-decodable to `{success: true, transaction: 0x..., network, payer}`.
   - Read `usdc.balanceOf(vault)` before and after; assert it increased by `value`.

- **Anvil spawn helper (`waitForPort`):**
  ```ts
  import net from 'node:net';
  async function waitForPort(port: number, opts: { timeout: number, host?: string }) {
    const host = opts.host ?? '127.0.0.1';
    const deadline = Date.now() + opts.timeout;
    while (Date.now() < deadline) {
      const ok = await new Promise<boolean>((resolve) => {
        const sock = net.connect({ port, host });
        sock.once('connect', () => { sock.end(); resolve(true); });
        sock.once('error', () => resolve(false));
      });
      if (ok) return;
      await new Promise((r) => setTimeout(r, 250));
    }
    throw new Error(`waitForPort: ${host}:${port} did not become ready within ${opts.timeout}ms`);
  }
  ```
