# Task 8: Core orchestrator + adapters + index

## Required Skills

Before starting, load:
- `/skill:code-writing` — [skills/code-writing/SKILL.md](~/.claude/skills/code-writing/SKILL.md)

## Description

Wire the middleware's facilitator pipeline together. After Task 7 produced `verify.ts` (EIP-712 ecrecover + Solution-7c semantic checks + replay-store has+insert) and `settle.ts` (USDC.transferWithAuthorization writeContract with receipt-await and failure classification), this task glues them into the single request-time entry point and exposes the public API.

Four files:

1. **`core.ts`** — framework-agnostic orchestrator. Exports `paywall(req, opts)` where `req` is `{headers, method, url}` and `opts: PaywallConfig`. Runs the Solution "How it works" pipeline (steps 7a–7g) end-to-end: read X-PAYMENT, enforce the 4 KB size cap (D18 `header_too_large`), decode (`malformed_payment_header`), call `verify`, perform off-chain factory-state checks against the 5-second cache (paused / vault deployed — Decisions row D12 + Shared resources row 4), call `settle`, emit X-PAYMENT-RESPONSE on success, return either a 402 body or a passthrough decision.

   **Resource ownership (per systemic-fix §5 / D13):**
   - Owns the lazy-init `PublicClient` per network (cached across requests).
   - Owns the process-singleton `NonceStore` (module-scope, not closure-scope).
   - Owns the per-network `FactoryStateCache` (TTL 5 s).
   - Does **NOT** create or own `WalletClient`. WalletClient lives in `settle.ts` (T7).
   - Does **NOT** extract the underlying key from `OpaqueRelayerKey`. Core holds the `OpaqueRelayerKey` instance opaque and passes it through to `settle.ts` as a function argument. `settle.ts` is the only consumer that extracts via the private symbol.

   Wires `SecurityLogger` (D18) at every defined emit site — fire-and-forget, swallowing logger throws so a misconfigured logger never blocks the request. The D14 startup chainId pin lives in `settle.ts` (the first writeContract caller per request) per systemic-fix §5; `core.ts` does not perform the chainId assertion itself but its PublicClient may be used by `verify.ts` and `settle.ts` for reads.

2. **`adapters/node-http.ts`** — `withPaywall(handler, opts): (req: IncomingMessage, res: ServerResponse) => Promise<void>`. Normalizes Node's `IncomingMessage` into the `{headers, method, url}` shape `core.paywall` accepts; when `paywall` returns a 402 result, writes status + headers + body to `res`; when it returns `passthrough`, sets the X-PAYMENT-RESPONSE header on `res` and invokes the user's `handler(req, res)`. Propagates exceptions from the user handler unchanged.

3. **`adapters/fastify.ts`** — `fastifyPaywall(opts): FastifyPluginAsync`. Registers a `preHandler` hook that calls `core.paywall` with the Fastify `request`'s normalized headers/method/url. On 402, the hook calls `reply.code(402).send(body)` (with any extra headers set via `reply.header(...)` first) and then `return reply` to short-circuit the lifecycle — Fastify treats `reply.send` from a preHandler hook as "the response has been sent; do not invoke the route handler". On passthrough, sets `X-PAYMENT-RESPONSE` via `reply.header(...)` and returns nothing (no `reply.send`), so Fastify continues to the route handler which sends 200.

4. **`index.ts`** — public ESM exports for the npm package. Exports `withPaywall`, `fastifyPaywall`, `NETWORKS`, the `SecurityLogger` interface, and the public types (`PaywallConfig`, `NetworkConfig`, `PaymentRequirements`, `PaymentPayload`, `ExactEvmPayload`, `OpaqueRelayerKey`). Does NOT re-export internal helpers (`verify`, `settle`, `replay-store` internals, the relayer-key symbol).

This task completes the middleware's runtime surface. After this, the package's `npm run build` produces a working `@universal-paywall/middleware` that can be imported and used by both adapters. Tests (Task 9) and integration (Task 10) come next.

## What to do

1. **Implement `core.ts`** — the orchestrator function `paywall(req, opts)` and its supporting state:
   - **PublicClient lazy-init per network** — build viem `PublicClient` from `NETWORKS[opts.network].rpcUrl` (or `opts.facilitator.rpcUrl` override if supplied). Cache by network key across requests in a module-scope `Map<string, PublicClient>`. First-call concurrency: guard with an in-flight `Promise<PublicClient>` cache so two simultaneous first requests share one init.
   - **Process-singleton `NonceStore`** — constructed once at module load. Truly module-scope (not closure-scope inside `paywall(...)`), so the same instance is observable across both adapters in the same process (cross-adapter replay rejection — Task 10 forked-e2e depends on this).
   - **Per-network `FactoryStateCache`** — small object holding `{paused: boolean, vaults: Map<eoa, address>, fetchedAt: number}` with 5 s TTL. On read, if stale → re-fetch `factory.paused()` and the specific `factory.vaults(developerEoa)` via `publicClient.readContract`. On RPC error: if the cached value is non-stale return it; otherwise propagate as a settlement-style failure (`settlement_failed` with `reason: 'rpc_5xx'`). `vaults(developerEoa)` once non-zero is final (factory deploys are immutable per D3); cache forever once non-zero. Re-fetch only while still `0x0`. Cache stampede on expiry: share one in-flight refresh via Promise dedup.
   - **OpaqueRelayerKey handling** — `core.ts` receives `opts.facilitator.relayerKey: OpaqueRelayerKey` and holds it opaque. When invoking `settle.ts`, pass the wrapper instance as-is. Do **NOT** import the private extract symbol from `relayer-key.ts`. Do **NOT** `JSON.stringify` it. Do **NOT** log it. The wrapper's own redaction hooks (`toJSON`, `toString`, `[util.inspect.custom]`) keep it safe from accidental serialization.
   - **Pipeline** — implement Solution "How it works" steps 7a–7g in order:
     - 7a: read `headers['x-payment']` (case-insensitive); if missing → return 402 with `build402Body(PaymentRequirements)` per D1, body's `error` field omitted or set to `payment_required` per systemic-fix §4. Size check: `Buffer.byteLength(headerValue, 'utf-8') <= 4096` else return 400 `header_too_large` + emit `header_too_large {size}`.
     - 7b: `decodeXPayment(headerValue)` (helper from `x402.ts`). On failure return 400 `malformed_payment_header` + emit `malformed_header {phase: 'base64' | 'json' | 'shape'}`.
     - 7c: call `verify(payload, opts, {publicClient, nonceStore})`. On failure return 402 with the reason returned by verify (one of `invalid_signature`, `insufficient_amount` with `required`/`received` extras, `authorization_expired`, `authorization_not_yet_valid`, `nonce_already_used`, `network_mismatch`, `to_mismatch` — see systemic-fix §4 for the canonical strings).
     - 7d: factory-state checks against the cache. If `factory.paused() === true` → 402 `paused` + emit `paused_request {developerEoaHash}`. If `factory.vaults(opts.developerEoa) === 0x0` → 402 `vault_not_deployed` + emit `vault_not_deployed {developerEoaHash}`.
     - 7e: call `settle(payload, opts, {publicClient, relayerKey: opts.facilitator.relayerKey})`. Pass the OpaqueRelayerKey as-is (settle owns the WalletClient and the extract). On success: get back `{txHash, network, payer}`. On classified failure: return 402 `settlement_failed` with `reason` ∈ `{rpc_timeout, rpc_5xx, gas_estimate_revert, mine_timeout, receipt_reverted, relayer_no_balance, authorization_already_used_onchain}` + emit `settlement_failed {payerHash, reason, txHash?}` (no raw signature, no relayer info).
     - 7f: on success, set X-PAYMENT-RESPONSE = `base64(JSON({success:true, transaction: txHash, network, payer: payload.payload.authorization.from}))` per D1.
     - 7g: return `{kind: 'passthrough', responseHeaders: {'X-PAYMENT-RESPONSE': ...}}`.
   - **Logger emit helper** — build a tiny `emit(name, payload)` helper inside `core.ts` that does `try { opts.logger?.securityEvent(name, scrubSecrets(payload)) } catch { /* swallow */ }`. Wrap each call individually so a throw on event N doesn't break events N+1 in the same handler. Default logger when `opts.logger` undefined is a no-op (don't even call the helper, or have the helper short-circuit when `opts.logger` is undefined).
   - **Hash helpers** — `payerHash(addr)` = first 10 chars of `keccak256(addr)` (i.e. `0x` + 8 hex chars), per D18 catalog. `developerEoaHash`, `nonceHash` analogously.
   - Return shape: a discriminated union — `{kind: '402', status: 402, headers: Record<string,string>, body: object}` OR `{kind: 'passthrough', responseHeaders: {'X-PAYMENT-RESPONSE': string}}`. Both adapters consume this shape.

2. **Implement `adapters/node-http.ts`** — `withPaywall(handler, opts)`:
   - Returns a function `(req: IncomingMessage, res: ServerResponse) => Promise<void>`.
   - Normalizes `req.headers` (Node's headers are already lowercased string-or-array; pick the first element if array), `req.method`, `req.url`.
   - Awaits `core.paywall(...)`. If `kind === '402'`: `res.writeHead(402, {'Content-Type': 'application/json', ...result.headers})` then `res.end(JSON.stringify(result.body))`. If `kind === 'passthrough'`: copy `responseHeaders` onto `res` via `res.setHeader(name, value)` **before** invoking `handler(req, res)` (per Solution: "flushes before user handler is invoked"). Then `await handler(req, res)`.
   - Re-throws exceptions from `handler` unchanged (do not swallow).

3. **Implement `adapters/fastify.ts`** — `fastifyPaywall(opts)`:
   - `import type { FastifyPluginAsync, FastifyRequest, FastifyReply } from 'fastify'` (type-only; `fastify` is a peer dep, not a runtime import).
   - Returns a `FastifyPluginAsync` that registers a `preHandler` hook (server-wide; the integrator scopes it via Fastify's encapsulation if they want per-route).
   - In the hook: build `{headers: request.headers as Record<string, string>, method: request.method, url: request.url}`; await `core.paywall(...)`.
   - **On `kind === '402'`** — set any extra headers via `reply.header(name, value)`, then call `reply.code(402).send(body)`, then `return reply` to short-circuit. In a Fastify preHandler hook, returning the reply after `reply.send(...)` instructs Fastify to skip the route handler. This is a single, non-contradictory instruction: do `reply.code(402).send(body)` and `return reply` — both are required.
   - **On `kind === 'passthrough'`** — call `reply.header('X-PAYMENT-RESPONSE', responseHeaders['X-PAYMENT-RESPONSE'])` and return nothing (no `reply.send`). Fastify proceeds to the route handler, which sends 200. Preserve reply chaining: the user route handler can still call `reply.send(...)` itself.

**Edge cases:**
- **Lazy-init concurrency for PublicClient**: two simultaneous first requests for the same network must not race-create two clients. Guard with a `Map<network, Promise<PublicClient>>` cache — second caller awaits the first's in-flight init promise.
- **Cache stampede on `FactoryStateCache` expiry**: at the moment of expiry, many requests should share one in-flight refresh via Promise dedup. The 5 s TTL is short enough that lock contention is rare, but still avoid N parallel RPC fetches.
- **`vaults(developerEoa)` caching**: once non-zero, the value is final (factory deploys are immutable per D3). Cache forever once non-zero. Re-fetch only while still `0x0`.
- **OpaqueRelayerKey ownership boundary**: `core.ts` MUST NOT import the private extract symbol from `relayer-key.ts`. Only `settle.ts` does. If you find yourself importing the symbol, you've broken the D13 boundary — pass the wrapper instead.
- **Logger try/catch granularity**: wrap each `logger.securityEvent(...)` call individually so a throw on event N doesn't break events N+1 in the same handler. Use the tiny `emit(name, payload)` helper.
- **402 body Content-Type**: `application/json`. Node http adapter MUST set this header on the 402 response (Solution step 3).
- **4 KB cap measurement**: `Buffer.byteLength(headerValue, 'utf-8')` is the correct measurement (UTF-8 byte length, not character length) — per tech-spec step 7a wording.
- **Header lookup**: case-insensitive. Node's `req.headers` is already lowercased; Fastify's `request.headers` is also lowercased by the framework. Look up via `headers['x-payment']`.
