## Description

Wire the middleware's facilitator pipeline together. After Task 7 produced `verify.ts` (EIP-712 ecrecover + Solution-7c semantic checks + replay-store has+insert) and `settle.ts` (USDC.transferWithAuthorization writeContract with receipt-await and failure classification), this task glues them into the single request-time entry point and exposes the public API.

Four files:

1. **`core.ts`** ŌĆö framework-agnostic orchestrator. Exports `paywall(req, opts)` where `req` is `{headers, method, url}` and `opts: PaywallConfig`. Runs the Solution "How it works" pipeline (steps 7aŌĆō7g) end-to-end: read X-PAYMENT, enforce the 4 KB size cap (D18 `header_too_large`), decode (`malformed_payment_header`), call `verify`, perform off-chain factory-state checks against the 5-second cache (paused / vault deployed ŌĆö Decisions row D12 + Shared resources row 4), call `settle`, emit X-PAYMENT-RESPONSE on success, return either a 402 body or a passthrough decision. Owns the lazy-init singletons listed in `## Shared resources` of tech-spec: `PublicClient` per network, `WalletClient` per (network, relayerKey), the process-singleton `NonceStore`, and the per-network factory-state cache. Wires `SecurityLogger` (D18) at every defined emit site ŌĆö fire-and-forget, swallowing logger throws so a misconfigured logger never blocks the request. Performs the D14 startup chainId pin (calls `publicClient.getChainId()` once on first use and asserts it equals `NETWORKS[network].chainId`, emits `chain_id_mismatch` and throws `NetworkMismatchError` from `errors.ts` on first request when wrong).

2. **`adapters/node-http.ts`** ŌĆö `withPaywall(handler, opts): (req: IncomingMessage, res: ServerResponse) => Promise<void>`. Normalizes Node's `IncomingMessage` into the `{headers, method, url}` shape `core.paywall` accepts; when `paywall` returns a 402 result, writes status + headers + body to `res`; when it returns `passthrough`, sets the X-PAYMENT-RESPONSE header on `res` and invokes the user's `handler(req, res)`. Propagates exceptions from the user handler unchanged.

3. **`adapters/fastify.ts`** ŌĆö `fastifyPaywall(opts): FastifyPluginAsync`. Registers a `preHandler` hook that calls `core.paywall` with the Fastify `request`'s normalized headers/method/url. On 402, calls `reply.code(402).headers(...).send(body)` and returns (so Fastify short-circuits the lifecycle); on passthrough, sets the X-PAYMENT-RESPONSE response header via `reply.header(...)` and returns (Fastify continues to the route handler, which sends 200). Preserves Fastify reply chaining on the 200 path (does NOT call `reply.send` itself on passthrough ŌĆö only mutates headers).

4. **`index.ts`** ŌĆö public ESM exports for the npm package. Exports `withPaywall`, `fastifyPaywall`, `NETWORKS`, the `SecurityLogger` interface, and the public types (`PaywallConfig`, `NetworkConfig`, `PaymentRequirements`, `PaymentPayload`, `ExactEvmPayload`, `OpaqueRelayerKey`). Does NOT re-export internal helpers (`verify`, `settle`, `replay-store` internals, the relayer-key symbol).

This task completes the middleware's runtime surface. After this, the package's `npm run build` produces a working `@universal-paywall/middleware` that can be imported and used by both adapters. Tests (Task 9) and integration (Task 10) come next.

**Dependencies:**
- Task 7 must be complete (delivers `verify.ts`, `settle.ts`).
- Implicitly depends on Task 6 (pure modules: `types.ts`, `errors.ts`, `x402.ts`, `relayer-key.ts`, `replay-store.ts`, `networks.ts`).
- Runtime: `viem` 2.x for `PublicClient.readContract`, `WalletClient` (via settle), `keccak256` for hashing in event payloads.
- Peer dep: `fastify` (type-only import in `adapters/fastify.ts`).

**Edge cases:**
- Lazy-init concurrency: two simultaneous first requests must not race-create two `PublicClient`s. Guard with a `Promise` cache (the second caller awaits the first's in-flight init promise).
- D14 chainId pin: must fire on FIRST request, not at module load ŌĆö `PublicClient.getChainId()` is async. Cache the result per network so subsequent requests pay no RPC cost.
- Cache stampede on `FactoryStateCache` expiry: if many requests hit at the moment of expiry, they should share one in-flight refresh (Promise dedup). The 5 s TTL is short enough that lock contention is rare, but still avoid N parallel RPC fetches.
- `developerEoa` is per-config, not per-request ŌĆö but vault lookup is per-developer. Cache `factory.vaults(eoa)` keyed by EOA. Once non-zero, the value is final (factory deploys are immutable per D3); MAY cache forever once non-zero. Re-fetch only while still `0x0`.
- `OpaqueRelayerKey` extraction: the WalletClient init is the ONE place that extracts via the private symbol (per D13: "settle.ts is the only consumer"). Confirm scope ŌĆö if settle owns extraction, core's WalletClient construction must hand off the wrapper, not extract directly. Verify against actual `relayer-key.ts` and `settle.ts` from Tasks 6/7.
- Logger try/catch must be tight per emit site ŌĆö wrap each `logger.securityEvent(...)` call individually so a throw on event N doesn't break events N+1 in the same handler.
- 402 body Content-Type is `application/json` (Solution step 3); adapter must set this even when only writing headers via `core`.
- `Buffer.byteLength` is the correct measurement for the 4 KB cap (utf-8 byte length, not character length) ŌĆö per tech-spec step 7a wording.
- Node http adapter MUST set `Content-Type: application/json` on the 402 response (Solution step 3).

**Implementation hints:**
- The Solution pipeline reads top-to-bottom as the orchestrator function body ŌĆö implement it in that order so the source mirrors the spec.
- Build a tiny helper `emit(name, payload)` inside `core.ts` that does `try { opts.logger?.securityEvent(name, scrubSecrets(payload)) } catch {}`. Use it at every emit site so there's exactly one redaction/swallow location.
- Hash helpers: `payerHash(addr) = keccak256(addr).slice(0, 10)` (i.e., first 8 hex chars after `0x`) per D18 catalog. Add `developerEoaHash`, `nonceHash` analogously. Centralize in `core.ts` (or a tiny `event-payloads.ts` helper if it grows).
- For the Fastify reply-chaining test: write a Fastify route that returns `{ok:true}` and assert the test client receives `{ok:true}` + the X-PAYMENT-RESPONSE header in the same response.
- For the cross-adapter NonceStore proof (Task 10's forked-e2e will need it): keep `NonceStore` truly module-scope, not closure-scope inside `paywall(...)`.
- `viem.PublicClient.readContract` for `factory.paused()` and `factory.vaults(developerEoa)` ŌĆö ABI lives in `networks.ts` or a small `factory-abi.ts` if not already there. Confirm with the actual Task 6 deliverable.
- `index.ts` must not `import './core.js'` unless it re-exports something from it (tree-shake) ŌĆö keep imports minimal so the bundle stays <30 KB minified+gzip per the AC list.
- ESM-only: every relative import has a `.js` extension (NodeNext resolver requirement) ŌĆö including in the adapter sub-folder (`../core.js`).
