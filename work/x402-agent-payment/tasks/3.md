# Task 3: Verify Arc Testnet USDC supports EIP-3009 + measure gas (spike)

## Required Skills

Перед выполнением задачи загрузи:
- `/skill:code-writing` — [skills/code-writing/SKILL.md](~/.claude/skills/code-writing/SKILL.md)

## Description

Wave 2 spike: write a Hardhat TypeScript script that connects to live Arc Testnet RPC and de-risks two unknowns before we commit to Arc as the MVP chain.

1. **EIP-3009 surface check.** Confirm that the canonical Arc Testnet USDC contract (`0x3600000000000000000000000000000000000000`) exposes the methods our facilitator depends on:
   - `transferWithAuthorization(address,address,uint256,uint256,uint256,bytes32,uint8,bytes32,bytes32)` — selector **`0xe3ee160e`** (NOT `0xef55bec6`, which is the unrelated `receiveWithAuthorization` selector — checking the wrong one would give a false positive)
   - `authorizationState(address,bytes32)` — required by `verify.ts` to detect on-chain replay
   - `name()`, `version()`, `decimals()` — EIP-712 domain inputs and ERC-20 view

2. **Gas economics check.** Estimate gas for a sample `transferWithAuthorization` call against the live contract, convert the estimate to USDC cost using current `gasPrice` (Arc Testnet pays gas in USDC), and surface a warning to the operator if `gasCost > 0.0005 USDC` (5% of a 0.01 USDC payment — the per-payment economics threshold from `external-analysis.md`). This is the empirical input to Risks-table row "Per-payment settlement creates per-event gas overhead".

3. **Emit JSON artifact for Task 6.** Write the verified `name()`, `version()`, `decimals()`, `supportsEip3009`, and gas measurement to `contracts/scripts/arc-testnet-usdc-domain.json`. Task 6 (Wave 5, `networks.ts` author) reads this artifact to populate `usdcEip712Name` and `usdcEip712Version` in the NETWORKS registry. **This JSON artifact is the only handoff out of this task** — do NOT touch `packages/middleware/src/networks.ts` directly (it does not exist yet at this point in the wave sequence; Task 6 owns its creation).

The script is run manually by the operator (and by the agent during `Verify-smoke`); it is NOT part of the automated test suite. Its outputs land in the tech-spec deviation log if the live contract diverges from assumptions, and we pivot to Base Sepolia (per Risks fallback) if `supportsEip3009 === false`.

## What to do

1. **Confirm Task 2's Hardhat Arc Testnet network config is present.** Read `contracts/hardhat.config.ts` (created in Task 2). Confirm the `arcTestnet` network entry exists with `chainId: 5042002`, `url` from `ARC_RPC_URL` env var (default fallback `https://rpc.testnet.arc.network`), and `accounts` from `DEPLOYER_KEY` (optional — the spike only does reads + gas estimation, no signing required). If anything is missing, surface a deviation note rather than silently patching Task 2's output.

2. **Create `contracts/scripts/verify-usdc-eip3009.ts`.** The script must:
   - Use `hre.ethers` (hardhat-toolbox already provides ethers) — no need to pull in `viem` here since the rest of `contracts/` is ethers-based.
   - Hardcode the USDC address `0x3600000000000000000000000000000000000000` as a `const USDC_ADDRESS`.
   - Construct a minimal ABI fragment covering: `name()`, `version()`, `decimals()`, `transferWithAuthorization(...)`, `authorizationState(address,bytes32)`.
   - Call `name()`, `version()`, `decimals()` and record results.
   - **Selector self-check:** assert the locally-computed selector for `transferWithAuthorization(address,address,uint256,uint256,uint256,bytes32,uint8,bytes32,bytes32)` equals `0xe3ee160e`. This guards against a typo in the ABI string.
   - **On-chain method existence:** prove that both `authorizationState(address,bytes32)` and `transferWithAuthorization(...)` are present on the live contract. Choose a reliable detection strategy for the JSON-RPC at hand — the script's contract is to set `supportsEip3009 = false` only when there is unambiguous evidence the method does not exist, and `true` when the method clearly does exist. Ambiguous outcomes must surface as a `notes` entry, not a silent assumption.
   - **Gas estimation:** attempt a gas estimate for a sample `transferWithAuthorization` call with dummy arguments (random nonce, plausible `validBefore`, sample `value = 10000n` = 0.01 USDC in micro-USDC, dummy signature bytes). If the node refuses to estimate gas for a reverting call, fall back to a documented constant lower-bound and tag the JSON output to make the fallback visible.
   - **Convert gas to USDC cost:** read `gasPrice` from `provider.getFeeData()`, compute `gasCostWei = gasEstimate * gasPrice`, and convert to micro-USDC. Be explicit in code about which decimal scale each variable lives in (Arc reports gas in wei-scale but `decimals()` is 6 for the ERC-20 view — the dual-decimal foot-gun flagged in tech-spec line 208). If the math feels off (cost looks 10^12× too small or too large), STOP and surface to operator — do NOT publish a wrong number.
   - **Pretty-print the result** as JSON on stdout (matches `Verify-smoke` expectation): `{name, version, decimals, supportsEip3009, sampleGasCost: "<n> micro-USDC", gasCostExceedsThreshold: boolean, notes?: string[]}`.
   - **Exit non-zero if `supportsEip3009 === false` or `decimals !== 6`** — these are hard blockers per Risks rows.
   - **Print a clearly-formatted warning (but exit 0)** if `gasCostExceedsThreshold === true` — that signals the per-payment economics need a post-MVP batched-settlement follow-up but does NOT block the wave.

3. **Write `contracts/scripts/arc-testnet-usdc-domain.json`** with the same shape as the stdout JSON. This is the sole handoff to Task 6 — Task 6 reads this file to populate `NETWORKS['arc-testnet'].usdcEip712Name` and `NETWORKS['arc-testnet'].usdcEip712Version` (and the canonical `eip155:5042002` mirror). Do NOT modify `packages/middleware/src/networks.ts` — it does not exist at this wave and Task 6 owns its creation. The script header must call out this dataflow so Task 6 picks up the artifact.

4. **Document spike outcome in `work/x402-agent-payment/decisions.md`** post-completion: the four chain-read values (name/version/decimals/supportsEip3009), the measured gas cost, whether it exceeds the 5% threshold, and any dual-decimal surprises.

**Dependencies:**

- Wave 2 follows Wave 1 (T1, T2). T4 runs in parallel waves but does not depend on this task; Task 6 (Wave 5) consumes the JSON artifact this task produces.

- Env vars: `ARC_RPC_URL` (required, defaults to `https://rpc.testnet.arc.network`); `DEPLOYER_KEY` is NOT required by this task — the script is read-only and does not sign.

**Edge cases:**
- RPC unreachable / 5xx / timeout: script should retry up to 3 times with backoff, then exit non-zero with a clear "RPC unavailable" message (not a stack trace dump — operator-facing UX).
- `gasPrice === 0n` from `getFeeData()` (some Arc dev configurations): do NOT silently report `0 micro-USDC`. Surface "gas pricing unavailable, threshold check skipped" via the `notes` field and skip the threshold breach assertion.
- Live USDC contract reverts `name()` / `version()` (rare for a USDC-compliant contract, but theoretically possible): treat as hard failure and exit non-zero — the EIP-712 domain construction depends on these.
- Dual-decimal interaction: Arc treats USDC as native gas. `gasPrice` may be reported in 18-decimal wei-equivalent while `decimals()` reports 6 for ERC-20 view. Be explicit in the cost conversion code about which decimal scale each variable is in. If the math feels off, STOP and surface to operator — do NOT publish a wrong number.
- Selector self-check: if the computed selector doesn't equal `0xe3ee160e`, the ABI signature string in the script is wrong — fail loudly with a developer-facing error, not a chain check.

**Implementation hints:**
- Use a typed `interface UsdcEip3009Probe` for the result object — don't return `any`.
- The JSON artifact must be written using `fs.writeFileSync` with `JSON.stringify(result, null, 2)` so a human can diff it.
- Per `patterns.md`, run the workspace lint command before declaring done.
- Per `patterns.md` security gate: gitleaks must not flag the spike script. The script must not write the relayer key or any 0x-hex 64-char string to stdout / file unless it's the expected USDC address.
- The script header should include a comment block: source RPC, target USDC address, intended consumer of output (Task 6 `networks.ts` via `arc-testnet-usdc-domain.json`), and a pointer to this task file.
- Wave sequencing note: this task ships in Wave 2; `packages/middleware/src/networks.ts` is created in Task 6 (Wave 5). The JSON artifact is the only handoff — do NOT touch `networks.ts` from this task.
