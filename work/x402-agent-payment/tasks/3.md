# Task 3: Verify Arc Testnet USDC supports EIP-3009 + measure gas (spike)

## Required Skills

Перед выполнением задачи загрузи:
- `/skill:code-writing` — [skills/code-writing/SKILL.md](~/.claude/skills/code-writing/SKILL.md)

## Description

Wave 1 spike: write a Hardhat TypeScript script that connects to live Arc Testnet RPC and de-risks two unknowns before we commit to Arc as the MVP chain.

1. **EIP-3009 surface check.** Confirm that the canonical Arc Testnet USDC contract (`0x3600000000000000000000000000000000000000`) exposes the methods our facilitator depends on:
   - `transferWithAuthorization(address,address,uint256,uint256,uint256,bytes32,uint8,bytes32,bytes32)` — selector **`0xe3ee160e`** (NOT `0xef55bec6`, which is the unrelated `receiveWithAuthorization` selector — checking the wrong one would give a false positive)
   - `authorizationState(address,bytes32)` — required by `verify.ts` to detect on-chain replay
   - `name()`, `version()`, `decimals()` — EIP-712 domain inputs and ERC-20 view

2. **Gas economics check.** Estimate gas for a sample `transferWithAuthorization` call against the live contract, convert the estimate to USDC cost using current `gasPrice` (Arc Testnet pays gas in USDC), and surface a warning to the operator if `gasCost > 0.0005 USDC` (5% of a 0.01 USDC payment — the per-payment economics threshold from `external-analysis.md`). This is the empirical input to Risks-table row "Per-payment settlement creates per-event gas overhead".

3. **Patch `NETWORKS` config.** The Task 6 `networks.ts` ships with `arc-testnet.usdcEip712Name` and `arc-testnet.usdcEip712Version` stubbed. After this spike reads the real values from chain, patch `packages/middleware/src/networks.ts` (both `arc-testnet` alias entry and `eip155:5042002` canonical entry) with the verified values.

The script is run manually by the operator (and by the agent during `Verify-smoke`); it is NOT part of the automated test suite. Its outputs land in the tech-spec deviation log if the live contract diverges from assumptions, and we pivot to Base Sepolia (per Risks fallback) if `supportsEip3009 === false`.

## What to do

1. **Set up Hardhat Arc Testnet network config consumption.** Read `contracts/hardhat.config.ts` (created in Task 2). Confirm the `arcTestnet` network entry is present with `chainId: 5042002`, `url` from `ARC_TESTNET_RPC_URL` env var (default fallback `https://5042002.rpc.thirdweb.com`), and `accounts` from `ARC_TESTNET_PRIVATE_KEY` (optional — the spike only does reads + gas estimation, no signing required). If anything is missing, surface a deviation note rather than silently patching Task 2's output.

2. **Create `contracts/scripts/verify-usdc-eip3009.ts`.** The script must:
   - Use `hre.ethers` (hardhat-toolbox already provides ethers) — no need to pull in `viem` here since the rest of `contracts/` is ethers-based.
   - Hardcode the USDC address `0x3600000000000000000000000000000000000000` as a `const USDC_ADDRESS`.
   - Construct a minimal ABI fragment covering: `name()`, `version()`, `decimals()`, `transferWithAuthorization(...)`, `authorizationState(address,bytes32)`.
   - Call `name()`, `version()`, `decimals()` and record results.
   - **Selector check (preferred path):** compute `id("transferWithAuthorization(address,address,uint256,uint256,uint256,bytes32,uint8,bytes32,bytes32)").slice(0,10)` and assert it equals `0xe3ee160e`. This is a self-check that our ABI string is correct.
   - **On-chain method existence:** call `authorizationState(0x0..., 0x0...)` — a pure view that should return `false` for a zero authorization. If the call reverts with `function selector not recognized` (or similar), set `supportsEip3009 = false`. Likewise, do a `staticCall` to `transferWithAuthorization(...)` with all-zero args — expect either a revert with a domain-specific error (`InvalidSignature`, `AuthorizationExpired`, etc.) which confirms the function exists, OR a "function not found" revert which confirms it doesn't. Distinguish the two by error string. (Both outcomes are acceptable signals; only "function not found" sets `supportsEip3009 = false`.)
   - **Gas estimation:** use `provider.estimateGas` against `transferWithAuthorization` with realistic args (random nonce, `validBefore = now + 600`, `value = 10000n` = 0.01 USDC in micro-USDC, dummy signature bytes). Catch the revert (expected, since the signature is invalid), but Arc nodes typically return a `gasUsed` estimate even for reverts via `eth_estimateGas` — if not, fall back to `eth_call` with `gas` tracing or a constant lower-bound estimate of `~80_000` gas with a clear "fallback estimate" flag.
   - **Convert gas to USDC cost:** `gasPrice = provider.getFeeData().gasPrice`; `gasCostWei = gasEstimate * gasPrice`. On Arc, gas is denominated in USDC (1 USDC = 10^18 wei-equivalent for the gas accounting, but 6-decimal for ERC-20 — confirm both representations during the spike). Compute `gasCostMicroUsdc` as USDC-denominated cost. If the dual-decimal handling is unclear from the live contract behavior, surface that to the operator (this is the "dual-decimal foot-gun" called out in Tech-Spec line 208).
   - **Pretty-print the result** as JSON on stdout (matches `Verify-smoke` expectation): `{name, version, decimals, supportsEip3009, sampleGasCost: "<n> micro-USDC", gasCostExceedsThreshold: boolean, notes?: string[]}`.
   - **Exit non-zero if `supportsEip3009 === false` or `decimals !== 6`** — these are hard blockers per Risks rows.
   - **Print a clearly-formatted warning (but exit 0)** if `gasCostExceedsThreshold === true` — that signals the per-payment economics need a post-MVP batched-settlement follow-up but does NOT block Wave 1.

3. **Patch `packages/middleware/src/networks.ts`** (created in Task 6, but for Wave 1 sequencing it may not exist yet — see Implementation hints) with the values read from chain:
   - `NETWORKS['arc-testnet'].usdcEip712Name = "<value from name()>"` 
   - `NETWORKS['arc-testnet'].usdcEip712Version = "<value from version()>"`
   - Same updates on the canonical `NETWORKS['eip155:5042002']` entry.
   - **If `networks.ts` does not exist yet (Task 6 ships after this task in topological order — wave 4 vs wave 1):** instead of patching the file, emit the verified values to a file at `contracts/scripts/arc-testnet-usdc-domain.json` so Task 6 can read them when it writes `networks.ts`. Note this deviation in `decisions.md` post-completion.

4. **Document spike outcome in `work/x402-agent-payment/decisions.md`** post-completion: the four chain-read values (name/version/decimals/supportsEip3009), the measured gas cost, whether it exceeds the 5% threshold, and any dual-decimal surprises.

## Verification Steps

### Smoke

```bash
cd contracts && npx hardhat run scripts/verify-usdc-eip3009.ts --network arcTestnet
```

Expected stdout (JSON line, exact field set):
```json
{"name": "USD Coin", "version": "2", "decimals": 6, "supportsEip3009": true, "sampleGasCost": "<n> micro-USDC"}
```

Failure modes to watch for:
- `supportsEip3009: false` — STOP, surface to operator, evaluate Base Sepolia fallback.
- `decimals !== 6` — STOP, dual-decimal foot-gun is real, D4 math invalid.
- `sampleGasCost > 500 micro-USDC` (= 0.0005 USDC = 5% of 0.01) — WARNING but not blocking; document in decisions.md and defer batched-settlement work to post-MVP.

**Dependencies:**
- Task 2 must be complete: `contracts/hardhat.config.ts` exists with `arcTestnet` network configured. This task does not block Task 4 (which only needs `hardhat compile`), but it informs `networks.ts` defaults in Task 6.
- No npm packages added — uses `hardhat`, `hardhat-toolbox`, `ethers` already present from Task 2.
- Env vars: `ARC_TESTNET_RPC_URL` (required, defaults to `https://5042002.rpc.thirdweb.com`); `ARC_TESTNET_PRIVATE_KEY` (NOT required — script is read-only).

**Edge cases:**
- RPC unreachable / 5xx / timeout: script should retry up to 3 times with backoff, then exit non-zero with a clear "RPC unavailable" message (not a stack trace dump — operator-facing UX).
- `gasPrice === 0n` from `getFeeData()` (some Arc dev configurations): use a fallback constant equal to mainnet-norm USDC-gas pricing or skip the cost conversion and surface "gas pricing unavailable, threshold check skipped" — do NOT silently report `0 micro-USDC`.
- Live USDC contract reverts `name()` / `version()` (rare for a USDC-compliant contract, but theoretically possible): treat as hard failure and exit non-zero — the EIP-712 domain construction depends on these.
- Dual-decimal interaction: Arc treats USDC as native gas. `gasPrice` may be reported in 18-decimal wei-equivalent while `decimals()` reports 6 for ERC-20 view. Be explicit in the cost conversion code about which decimal scale each variable is in. If the math feels off (e.g., gas cost looks 10^12× too small or too large), STOP and surface to operator — do NOT publish a wrong number.
- Function-existence detection: `eth_call` against a non-existent selector typically returns `0x` (empty) rather than reverting on EVM nodes. The script should treat `0x`-return on `authorizationState(0, 0)` as "function missing", and a non-empty return (even `0x00...00`) as "function exists".
- Selector self-check robustness: if `id("transferWithAuthorization(...)").slice(0,10)` doesn't equal `0xe3ee160e`, the ABI signature string in the script is wrong — fail loudly with a developer-facing error, not a chain check.

**Implementation hints:**
- For the `transferWithAuthorization` gas estimation, build a dummy `bytes32 nonce = randomBytes(32)`, `uint8 v = 27`, `bytes32 r = 0x00…01`, `bytes32 s = 0x00…01`. The estimate will revert at signature recovery but ethers/hardhat surfaces a gas estimate prior to the revert — use that. If the RPC refuses to estimate-gas for reverting calls, fall back to a static lower-bound (`~80_000`) and annotate the JSON output `"sampleGasCostNote": "estimate is lower-bound fallback"`.
- Use a typed `interface UsdcEip3009Probe` for the result object — don't return `any`.
- Per `patterns.md`, run `npm run lint --workspace=contracts` (or whatever the contracts lint cmd is) before declaring done.
- Per `patterns.md` security gate: gitleaks must not flag the spike script. The script must not write the relayer key or any 0x-hex 64-char string to stdout / file unless it's the expected USDC address.
- Wave sequencing note: this task ships in Wave 1; `packages/middleware/src/networks.ts` is created in Task 6 (Wave 4). When Task 3 runs, `networks.ts` won't exist. Use the JSON-artifact fallback for Wave 1 and let Task 6 read the artifact. Document this dataflow clearly in the spike script's header comment so Task 6 picks it up.
- The script header should include a comment block: source RPC, target USDC address, intended consumer of output (Task 6 `networks.ts`), and a pointer to this task file.
