## Description

Full-feature security audit of the x402-agent-payment implementation across the middleware (`packages/middleware/src/`), Solidity contracts (`contracts/contracts/`), developer-onboarding CLI (`scripts/register.ts`), the deploy script (`contracts/deploy/01_deploy_factory.ts`), and the Arc Testnet USDC verification spike (`contracts/scripts/verify-usdc-eip3009.ts`). This is part of the Audit Wave (Wave 10) and runs after the implementation waves (Wave 3 — middleware pure modules; Wave 5 — verify/settle; Wave 6 — core+adapters; Wave 7 — middleware unit tests; Wave 8 — deploy script + register CLI; Wave 9 — forked + Arc Testnet e2e) have landed. T13 runs in parallel with T12 (Code Audit) and T14 (Test Audit). The auditor is the review for this task — no separate reviewers are spawned.

**Coverage-gate ownership clarification:** T13 does NOT run the coverage suites that gate ≥95% branch / ≥85% line targets — those are produced by Task 5 (contract coverage via `npx hardhat coverage`) and Task 14 (test-quality audit which validates coverage adequacy). T13 READS the most recent coverage reports (the artifacts left by Task 5 contract coverage and Wave 7 Task 9 vitest `--coverage` runs) as one input among many, and re-runs the security-tagged subsets only to confirm they still pass against the audited code. Coverage-threshold failures themselves are Task 14's finding domain; T13 cites them only if a security-tagged branch is uncovered.

Scope is split into two independent surfaces:

1. **Middleware (OWASP Top 10 ASVS-flavored):**
   - **X-PAYMENT validation surface** — confirm strict shape (`{x402Version, scheme, network, payload: {signature, authorization}}`), 4 KB size cap enforced at byte boundary (`Buffer.byteLength`), malformed/oversize produces typed HTTP 400 with no echo of attacker-controlled data, any extra fields outside the canonical shape are rejected (extension-vector defense), and `to_mismatch` rejection (the authorization's `to` field MUST equal the developer's vault address — any mismatch produces a typed verify error and is logged via `securityEvent("to_mismatch")` per D18 catalog).
   - **`OpaqueRelayerKey` handling (D13)** — verify that the wrapper field is non-enumerable, `toJSON`/`toString`/`[util.inspect.custom]` all redact, the key never leaks through `console.log`, `JSON.stringify`, `util.inspect`, `structuredClone`, or `pino`/`winston` serialization. Verify `scrubSecrets` coverage on all four shapes from D13: `0x[0-9a-fA-F]{64}` (raw private key with prefix), `[0-9a-fA-F]{64}\b` case-insensitive word-boundary (env-shape no-prefix), `0x[0-9a-fA-F]{130}` (65-byte ECDSA signature), and the `OpaqueRelayerKey` runtime brand. Verify `scrubSecrets` is applied at security-logger boundaries AND error-handler boundaries (both call sites). Confirm `settle.ts` is the only consumer that extracts the underlying key via the private symbol.
   - **RPC trust + chainId pin (D14)** — verify `publicClient.getChainId()` is asserted against `NETWORKS[network].chainId` on first use of a network; mismatch throws `NetworkMismatchError` BEFORE any settle path runs; the assertion is per-network (not global). Confirm `facilitator.rpcUrl` override is documented as trust-required and that no signed transaction can be broadcast before the chainId check passes.
   - **Integer math / value parsing** — `value >= maxAmountRequired` uses `BigInt` compare (no float coercion, no string compare). Price parser rejects `'1.2345678'`, `'abc'`, `''`, `'-1'`, `'0'`, `'1e2'`, `' 1 '` and accepts canonical decimal strings only. No arithmetic uses untrusted-string-as-number.
   - **EIP-712 chain-replay** — domain pins `chainId` AND `verifyingContract` AND `name` AND `version`; tampering any of the four causes verify to fail. Confirm timing checks: `validBefore > now + 5_000 ms` and `validAfter <= now` are computed against `Date.now()` (not `performance.now()`) and use the documented 5 s safety margin.
   - **Replay-store invariants** — `has` and `insert` execute as a single synchronous block with NO `await` between them (TOCTOU defense per D5). On settlement failure, the NonceStore entry is NOT deleted (per Risks row) — confirm this behavior in code, and confirm the retry path returns `nonce_already_used` rather than re-attempting settle.
   - **`SecurityLogger` payload integrity (D18)** — every payload in the typed catalog (10 events) uses hashed identifiers (`payerHash = keccak256(from)[0..8]`, `developerEoaHash`, `nonceHash`) NOT raw addresses or raw nonces. Confirm logger calls are wrapped in `try/catch` so a throwing logger does not block the request path. Confirm `securityEvent` is fire-and-forget (not awaited). Confirm no raw signature, no relayer info, no PII appears in any payload.
   - **Header parsing — DoS surface** — base64 decode is bounded (4 KB pre-decode cap), JSON parse is bounded (decoded size also implicitly capped by the 4 KB pre-decode bound), no recursion-depth attack on JSON.
   - **Settlement failure taxonomy (D5/D7)** — all seven failure reasons (`rpc_timeout`, `rpc_5xx`, `gas_estimate_revert`, `mine_timeout`, `receipt_reverted`, `relayer_no_balance`, `authorization_already_used_onchain`) emit through `securityEvent("settlement_failed")` without leaking the relayer address, the raw error message, or the signature.

2. **Solidity contracts (focused threat model):**
   - **Reentrancy** — `PaymentVaultImpl.withdraw()` uses `nonReentrant` (OZ storage-based per D8). Re-entry via a malicious treasury (cycle: fee transfer → treasury fallback → vault.withdraw()) is blocked. `PaymentSplitterFactory.register()` is reentrancy-safe (CEI ordering OR `nonReentrant` — confirm at least one).
   - **Access control** — `Ownable2Step` correctly applied to factory; `setFeeBps`, `setPlatformTreasury`, `pause`, `unpause`, `transferOwnership` all `onlyOwner`. `vault.withdraw()` is `onlyDeveloper`. No public function on the vault writes to `developer` or `factory` slots.
   - **Event emissions** — `VaultDeployed(developer, vault)`, `FeeBpsUpdated(oldBps, newBps)`, `PlatformTreasuryUpdated(oldTreasury, newTreasury)`, `Paused`, `Unpaused`, `Withdrawal(developer, gross, fee)` — all critical state transitions emit. `indexed` keyword is used appropriately for filterable fields. Two-step ownership transfer emits OZ's standard events.
   - **`Pausable` correctness (D12)** — factory inherits `Pausable`; `register()` reverts `EnforcedPause` when paused; `vault.withdraw()` is INTENTIONALLY NOT paused (developers retain access to their funds). Confirm this matches D12 and that no `whenNotPaused` modifier accidentally landed on vault paths.
   - **CREATE2 collision resistance (D3)** — salt formula is `bytes32(uint256(uint160(msg.sender)))`; only ONE vault per developer EOA can be deployed (`AlreadyRegistered` revert on second call). Confirm `vaults[developer]` mapping is checked before `cloneDeterministic` to prevent shadowing. Verify off-chain `Clones.predictDeterministicAddress` matches on-chain `computeVaultAddress` byte-for-byte (read the test in `PaymentSplitterFactory.test.ts` to confirm the cross-component invariant test exists).
   - **`_disableInitializers()` guard (D15)** — `PaymentVaultImpl` constructor calls `_disableInitializers()`. The implementation contract cannot be `initialize`-d directly. NatSpec invariant `no_selfdestruct no_delegatecall single_initializer` is present.
   - **No `receive()` / `fallback()` (D16)** — neither function exists on `PaymentVaultImpl`. Native-asset `call{value:x}` reverts at EVM dispatch. The documented carve-outs (SELFDESTRUCT push, pre-deploy CREATE2 funding) are intentional and bounded.
   - **No setters for `developer` / `factory` (D17)** — neither `setDeveloper(address)` nor `setFactory(address)` selectors exist in the compiled vault ABI; no function writes to those storage slots after `initialize`. NatSpec invariant `no_setters_for_developer_or_factory` is present.
   - **EIP-712 chain-replay safety on settle** — the call to `USDC.transferWithAuthorization` passes `from` as an explicit argument per EIP-3009 (D2). The middleware ecrecovers the signature off-chain BEFORE settle, so a tampered `from` is caught at verify time. Confirm no path forwards an unverified payload to the on-chain call.
   - **SELFDESTRUCT / DELEGATECALL absence (D15 NatSpec invariant)** — grep contract sources for `selfdestruct`, `delegatecall`, raw `assembly { ... selfdestruct(...) ... }`, raw `assembly { ... delegatecall(...) ... }`. None should be present in `PaymentVaultImpl.sol` OR `PaymentSplitterFactory.sol` (factory uses `Clones.cloneDeterministic` which DOES emit CREATE2; that is expected and not a finding).
   - **Custom errors hygiene** — `NotDeveloper`, `AlreadyRegistered`, `InvalidFeeBps`, `ZeroAddress`, `NoBalance` — confirm each is used at the right call site; no `require(false, "string")` fallbacks that bloat bytecode.
