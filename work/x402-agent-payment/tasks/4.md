# Task 4: Factory + Vault contracts

## Required Skills

Перед выполнением задачи загрузи:
- `/skill:code-writing` — [skills/code-writing/SKILL.md](~/.claude/skills/code-writing/SKILL.md)

## Description

Реализовать Solidity-контракты, образующие on-chain ядро платёжного потока x402: `PaymentSplitterFactory` (фабрика, разворачивающая per-developer хранилища через EIP-1167 minimal proxy + CREATE2) и `PaymentVaultImpl` (пассивный USDC-приёмник со сплитом fee при `withdraw()`). К контрактам добавить вспомогательный интерфейс `IERC3009` (для off-chain ABI в middleware) и тестовый мок `MockUsdcEip3009` с рабочей реализацией EIP-3009 для использования в Hardhat-тестах и forked-e2e.

Архитектура и инварианты безопасности заданы Решениями D3 (per-developer vault через `Clones.cloneDeterministic`), D4 (split в `withdraw()`), D8 (Solidity ^0.8.20 + OpenZeppelin 5.x: `Ownable2Step`, `Pausable`, `ReentrancyGuard` (storage-based, НЕ Transient), `Initializable`, `Clones`, `SafeERC20`), D10 (configurable fee, 0..1000 bps, default 50 bps), D11 (settable `platformTreasury`, независимый от `owner`), D12 (`Pausable` проверяется off-chain middleware, vault.withdraw НЕ блокируется паузой), D15 (impl-контракт заблокирован от прямой инициализации через `_disableInitializers()`, без `selfdestruct`/`delegatecall`/assembly), D16 (нет `receive()`/`fallback()` payable), D17 (`developer` и `factory` пишутся ровно один раз в `initialize`, без сеттеров и миграционных хелперов). Эта задача — фундамент, на котором стоят все последующие волны (контрактные тесты, middleware verify/settle, deploy script, register CLI).

## What to do

1. **`contracts/contracts/interfaces/IERC3009.sol`** — минимальный Solidity-интерфейс EIP-3009 (используется только для off-chain ABI helper в middleware и для типизации мока). Сигнатуры:
   - `transferWithAuthorization(address from, address to, uint256 value, uint256 validAfter, uint256 validBefore, bytes32 nonce, uint8 v, bytes32 r, bytes32 s)`
   - `authorizationState(address authorizer, bytes32 nonce) returns (bool)`
   - Событие `AuthorizationUsed(address indexed authorizer, bytes32 indexed nonce)`.

2. **`contracts/contracts/PaymentVaultImpl.sol`** — реализация хранилища (clone target):
   - Наследует `Initializable`, `ReentrancyGuard` (storage-based, OpenZeppelin 5.x).
   - Хранилище: `address public developer`, `address public factory`. Поля устанавливаются ровно один раз в `initialize(address _developer)` (D17). Сеттеров нет.
   - Constructor: `_disableInitializers()` — блокирует прямую инициализацию impl-контракта (D15).
   - `initialize(address _developer)` помечена модификатором `initializer`; `factory` устанавливается из `msg.sender` (фабрика — единственный, кто вызывает initialize при cloneDeterministic). Revert `ZeroAddress` если `_developer == 0`.
   - `withdraw()` — `external nonReentrant`, доступна только когда `msg.sender == developer` (revert `NotDeveloper`). Алгоритм (порядок трансферов канонический по systemic-fix §7 + code-research.md — **developer FIRST, platform SECOND**):
     - `usdc = IERC20(IPaymentSplitterFactory(factory).usdc())` (читается из фабрики; см. D11/D4 — фабрика владеет ссылкой на USDC).
     - `gross = usdc.balanceOf(address(this))`; revert `NoBalance` если `gross == 0`.
     - `feeBps = IPaymentSplitterFactory(factory).feeBps()`; `fee = gross * feeBps / 10000`; `net = gross - fee`.
     - `SafeERC20.safeTransfer(usdc, developer, net);` — **developer FIRST**.
     - `if (fee > 0) { SafeERC20.safeTransfer(usdc, IPaymentSplitterFactory(factory).platformTreasury(), fee); }` — **platform SECOND**.
     - Emit `Withdrawal(developer, gross, fee)`.
   - Никаких `receive()` / `fallback()` (D16) — нативные `call{value:x}` отбиваются на уровне EVM dispatch.
   - NatSpec на контракте: `@custom:security-invariant no_selfdestruct no_delegatecall single_initializer no_setters_for_developer_or_factory` (объединяет D15 и D17 в одном заголовке для security-auditor).
   - Запрет на использование `selfdestruct`, `delegatecall`, любых assembly-блоков, синтезирующих эти опкоды.
   - Custom errors: `NotDeveloper()`, `NoBalance()`, `ZeroAddress()`.

3. **`contracts/contracts/PaymentSplitterFactory.sol`** — фабрика:
   - Наследует `Ownable2Step`, `Pausable` (OpenZeppelin 5.x).
   - Хранилище (точно по Data Models техспека):
     - `IERC20 public immutable usdc;`
     - `address public platformTreasury;`
     - `uint16 public feeBps;` (хард-кап 1000)
     - `address public immutable vaultImpl;`
     - `mapping(address developer => address vault) public vaults;`
   - Constructor `(address _usdc, address _vaultImpl, address _platformTreasury, uint16 _feeBps, address _initialOwner)` — устанавливает `Ownable(_initialOwner)`, проверяет zero-address на каждом аргументе (revert `ZeroAddress`), revert `InvalidFeeBps` если `_feeBps > 1000`.
   - `register()` — `external whenNotPaused`:
     - `if (vaults[msg.sender] != address(0)) revert AlreadyRegistered();`
     - `bytes32 salt = bytes32(uint256(uint160(msg.sender)));` (канонический salt из D3).
     - `address vault = Clones.cloneDeterministic(vaultImpl, salt);`
     - `vaults[msg.sender] = vault;`
     - `PaymentVaultImpl(vault).initialize(msg.sender);`
     - Emit `VaultDeployed(msg.sender, vault)`.
   - `computeVaultAddress(address developer) external view returns (address)` — возвращает `Clones.predictDeterministicAddress(vaultImpl, bytes32(uint256(uint160(developer))), address(this))`. Pure-функция от входа (используется off-chain middleware для `payTo`).
   - `setFeeBps(uint16 newBps) external onlyOwner` — revert `InvalidFeeBps` если `newBps > 1000`, emit `FeeBpsUpdated(oldBps, newBps)`.
   - `setPlatformTreasury(address newTreasury) external onlyOwner` — revert `ZeroAddress` если `newTreasury == 0`, emit `PlatformTreasuryUpdated(oldTreasury, newTreasury)`.
   - `pause() external onlyOwner` / `unpause() external onlyOwner` — наследуются через `Pausable._pause()/_unpause()`. Эмитятся стандартные OZ-события `Paused(account)` / `Unpaused(account)`.
   - События `VaultDeployed`, `FeeBpsUpdated`, `PlatformTreasuryUpdated` — индексация и сигнатуры точно по Data Models техспека.
   - Custom errors: `AlreadyRegistered()`, `InvalidFeeBps()`, `ZeroAddress()`.

4. **`contracts/contracts/mocks/MockUsdcEip3009.sol`** — тестовый мок USDC с EIP-3009:
   - Наследует `ERC20Permit` или эквивалент OpenZeppelin для готовой EIP-712 domain separator; либо реализовать EIP-712 вручную (по аналогии с реальным USDC).
   - Реализует `IERC3009`: `transferWithAuthorization(...)` + `authorizationState(address, bytes32)` + событие `AuthorizationUsed`.
   - Проверки внутри `transferWithAuthorization`:
     - `block.timestamp > validAfter` (revert "AuthExpired" или аналогично).
     - `block.timestamp < validBefore`.
     - `!authorizationState[from][nonce]`.
     - `ecrecover` по EIP-712 domain `{name: "USD Coin", version: "2", chainId, verifyingContract: address(this)}` и типу `TransferWithAuthorization(address from,address to,uint256 value,uint256 validAfter,uint256 validBefore,bytes32 nonce)`.
     - Помечает `authorizationState[from][nonce] = true`, делает `_transfer(from, to, value)`, эмитит `AuthorizationUsed`.
   - Helper `mint(address to, uint256 amount)` без access control (только для тестов).
   - `decimals()` возвращает `6` (соответствует реальному USDC и спайку из Task 3).
   - В NatSpec пометить: `@dev Test-only mock — NOT for production deployment.`

5. **`@custom:security-invariant` NatSpec на impl-контракте** должен быть размещён над объявлением `contract PaymentVaultImpl is Initializable, ReentrancyGuard` так, чтобы slither / `forbidden-opcodes` lint могли якориться на нём (D15/D17).

6. После реализации убедиться, что `cd contracts && npx hardhat compile` проходит без warnings (или, если warnings неизбежны — задокументировать их в self-review).

**Edge cases:**
- `register()` дважды от того же EOA — должна быть ровно одна revert-ошибка `AlreadyRegistered`. Не позволять "перезаписать" mapping.
- `feeBps = 0` — `fee = 0`, второй transfer не делается (экономия gas + не эмитим лишний ERC20 Transfer event).
- `gross = 1` micro-USDC и `feeBps = 50` → `fee = 0` (целочисленное усечение), вся 1 unit уходит developer'у.
- `setFeeBps(1000)` — РАЗРЕШЕНО (хард-кап на верхней границе включителен).
- `setFeeBps(1001)` — revert `InvalidFeeBps`.
- `register()` когда `paused()` — revert OZ 5.x `EnforcedPause()`. Использовать модификатор `whenNotPaused` от `Pausable`.
- `withdraw()` когда `paused()` фабрики — должна работать (vault не проверяет паузу, D12). Это важная пограничная семантика — её документирует комментарий в коде vault.
- Salt: `bytes32(uint256(uint160(msg.sender)))` — точное приведение. НЕ использовать `keccak256(abi.encode(msg.sender))` или другие схемы, иначе off-chain `computeVaultAddress` не сойдётся с middleware (Task 6 будет считать тем же способом).

**Implementation hints (НЕ псевдокод, ориентиры):**
- В фабрике после `Clones.cloneDeterministic` нужно вызвать `initialize(msg.sender)` на свежем клоне. Поскольку клон делает DELEGATECALL в impl, состояние пишется в storage клона — `developer = _developer`, `factory = msg.sender` (т.е. адрес фабрики).
- В `PaymentVaultImpl.initialize` устанавливать `factory = msg.sender` — это автоматически означает, что инициализатором может быть только фабрика (developer.eoa никогда не вызовет initialize напрямую, потому что `_disableInitializers()` на impl + клон инициализируется фабрикой первым).
- Storage-based `ReentrancyGuard` (НЕ Transient): импорт `@openzeppelin/contracts/utils/ReentrancyGuard.sol`. На Arc Testnet EIP-1153 не верифицирован, и Transient-вариант может рассыпаться (D8).
- `Pausable._pause()` / `_unpause()` — internal; обёртку наружу делает фабрика:
  - `function pause() external onlyOwner { _pause(); }`
  - `function unpause() external onlyOwner { _unpause(); }`
- `Ownable2Step` — конструктор принимает `address initialOwner`. Двухшаговый transfer: `transferOwnership(newOwner)` (proposed) → `acceptOwnership()` (от newOwner). Это покрывает риск-row "owner-key compromise".
- При написании `MockUsdcEip3009`: реальный Circle USDC использует `name = "USD Coin"`, `version = "2"`. EIP-712 типы:
  - `EIP712Domain(string name,string version,uint256 chainId,address verifyingContract)`
  - `TransferWithAuthorization(address from,address to,uint256 value,uint256 validAfter,uint256 validBefore,bytes32 nonce)`
- `IPaymentSplitterFactory` mini-interface внутри `PaymentVaultImpl.sol` (или в `interfaces/`) — задаёт только `usdc()`, `feeBps()`, `platformTreasury()`. Это исключает циклический импорт фабрика↔vault.
- Все `address` поля в фабрике, имеющие `immutable`-семантику (`usdc`, `vaultImpl`), должны быть помечены `immutable` — это экономит SLOAD и фиксирует значение на этапе constructor.
- Никаких пользовательских `payable` функций нигде — даже в фабрике. Vault и фабрика не принимают ETH.
