## Description

Initialize the `contracts/` Hardhat TypeScript project that will host `PaymentSplitterFactory.sol` + `PaymentVaultImpl.sol` (Wave 3) and the Arc Testnet USDC spike script (Task 3). This task only sets up tooling and configuration ‚Äî no Solidity sources, tests, or deploy scripts are written here. After this task lands, any subsequent contract task can run `cd contracts && npx hardhat compile` against the configured Arc Testnet network and have all required OpenZeppelin 5.x primitives available.

Concrete deliverables:
- `contracts/package.json` declares Hardhat + toolbox + verify plugin + OpenZeppelin 5.x as deps, includes lint/compile/test scripts, and is wired into the npm workspace root.
- `contracts/hardhat.config.ts` pins Solidity `0.8.20` (with optimizer settings sane for prod), defines an `arcTestnet` network (chainId `5042002`, RPC `https://rpc.testnet.arc.network`, accounts read from env), and registers `@nomicfoundation/hardhat-toolbox` + `@nomicfoundation/hardhat-verify`.
- `contracts/tsconfig.json` defines a strict TS config aligned with the rest of the monorepo and includes `hardhat.config.ts`, `deploy/`, `scripts/`, and `test/`.
- Empty placeholder dirs (`contracts/contracts/`, `contracts/test/`, `contracts/scripts/`, `contracts/deploy/`) so `hardhat compile` does not error on missing folders.
- Root `package.json` workspaces array MUST list `contracts` (alongside `packages/*`, `apps/*`, `scripts` as established by Task 1).

Driven by tech-spec D8 (Solidity pragma + OZ 5.x choice), D9 (Hardhat over Foundry), architecture.md "Smart contracts" row, deployment.md canonical env-var names (`ARC_RPC_URL`, `DEPLOYER_KEY`), and Wave 1 task list.

## What to do

- Create the `contracts/` directory layout: `contracts/contracts/`, `contracts/test/`, `contracts/scripts/`, `contracts/deploy/`. Add a `.gitkeep` to each so the empty dirs are tracked.
- Write `contracts/package.json`:
  - `"name": "@universal-paywall/contracts"`, `"private": true`, `"type": "commonjs"` (Hardhat config requires CJS-flavored TS even when the rest of the monorepo is ESM; document the deliberate deviation from the root in a top-of-file comment in `hardhat.config.ts`).
  - `devDependencies`: `hardhat`, `@nomicfoundation/hardhat-toolbox`, `@nomicfoundation/hardhat-verify`, plus the typescript toolchain (`typescript`, `ts-node`, `@types/node`, `@types/chai`, `@types/mocha`), plus `dotenv` (read at top of `hardhat.config.ts`).
  - `dependencies`: `@openzeppelin/contracts@^5.0.0`.
  - Scripts: `compile` (`hardhat compile`), `test` (`hardhat test`), `coverage` (`hardhat coverage`), `lint` (typecheck ‚Äî `tsc --noEmit -p tsconfig.json`; Solidity linter / slither wiring is added in later tasks when source contracts exist).
- Write `contracts/hardhat.config.ts` in TypeScript:
  - `import "dotenv/config"` at the top so a local `contracts/.env` is honored.
  - Imports `@nomicfoundation/hardhat-toolbox` and `@nomicfoundation/hardhat-verify`.
  - `solidity` config pins version `0.8.20` (compatible with pragma `^0.8.20`), enables optimizer (`enabled: true`, `runs: 200`), `viaIR: false`.
  - `networks.arcTestnet`: `url: process.env.ARC_RPC_URL ?? "https://rpc.testnet.arc.network"`, `chainId: 5042002`, `accounts: process.env.DEPLOYER_KEY ? [process.env.DEPLOYER_KEY] : []` (empty array is valid for compile/test-only operation; deploy scripts require the env var). USE EXACTLY these env-var names ‚Äî `ARC_RPC_URL` and `DEPLOYER_KEY` per deployment.md.
  - `networks.hardhat`: default local network for tests (no extra config needed beyond toolbox defaults).
  - `etherscan.customChains` entry for `arcTestnet` with `{ network: "arcTestnet", chainId: 5042002, urls: { apiURL: "https://placeholder.example.com/api", browserURL: "https://placeholder.example.com" } }` plus a `// TODO(task-11): confirm arcscan API + browser URLs` comment. `etherscan.apiKey.arcTestnet = process.env.ARC_VERIFY_API_KEY ?? ""`. Do NOT invent fake URLs that look real ‚Äî `placeholder.example.com` is intentionally implausible so it fails loudly if used unmodified.
- Write `contracts/tsconfig.json`:
  - Inline strict settings (`"strict": true`, `"target": "es2022"`, `"module": "commonjs"` ‚Äî Hardhat TS config requires CJS, `"esModuleInterop": true`, `"resolveJsonModule": true`, `"skipLibCheck": true`, `"forceConsistentCasingInFileNames": true`).
  - `include`: `["hardhat.config.ts", "deploy/**/*.ts", "scripts/**/*.ts", "test/**/*.ts"]`.
- Edit root `package.json` (created by Task 1): ensure the `workspaces` array contains `"contracts"`. Task 1 owns the rest of the root file (lint, prettier, husky); this task's only edit there is appending `"contracts"` to `workspaces` if missing. If Task 1's `package.json` is not yet present at execution time, fail loudly rather than creating root config from this task.
- Add `contracts/.env.example` listing the canonical env-var names ‚Äî `ARC_RPC_URL`, `DEPLOYER_KEY`, `ARC_VERIFY_API_KEY` ‚Äî with placeholder values and a comment that the real `.env` is gitignored (gitleaks-blocked). Do NOT introduce `ARC_TESTNET_RPC_URL` or `ARC_TESTNET_PRIVATE_KEY` ‚Äî those names are forbidden by systemic decision ¬ß3.
- Add a trivial empty placeholder Solidity file `contracts/contracts/.gitkeep` only; do NOT add a placeholder `.sol`. Modern Hardhat (2.x) compiles cleanly with zero `.sol` files in the sources dir.

## Details

**Files:**
- `contracts/package.json` ‚Äî new. Workspace-private package, `"type": "commonjs"`, Hardhat dev deps + `@openzeppelin/contracts@^5.0.0` dep + `compile`/`test`/`coverage`/`lint` scripts. `dotenv` in devDependencies.
- `contracts/hardhat.config.ts` ‚Äî new. `import "dotenv/config"` first; Solidity 0.8.20 pinned with optimizer `{ enabled: true, runs: 200 }`; `arcTestnet` network using `ARC_RPC_URL` (default `https://rpc.testnet.arc.network`) + `DEPLOYER_KEY` (optional); toolbox + verify plugins imported; etherscan custom chain stub with `placeholder.example.com` URLs and `TODO(task-11)` comment.
- `contracts/tsconfig.json` ‚Äî new. Strict TS, CJS module (Hardhat requirement), includes config + deploy/scripts/test paths.
- `contracts/.env.example` ‚Äî new. Placeholders for `ARC_RPC_URL`, `DEPLOYER_KEY`, `ARC_VERIFY_API_KEY`. NO `ARC_TESTNET_*` variants.
- `contracts/{contracts,test,scripts,deploy}/.gitkeep` ‚Äî empty placeholders so the folders are tracked.
- `package.json` (root, from Task 1) ‚Äî append `"contracts"` to `workspaces` array if not present.

**Dependencies:**
- No task dependencies (Wave 1, parallel with Task 1).
- Task 1 owns the repo-root `package.json` skeleton (lint, prettier, husky). This task only appends `"contracts"` to its `workspaces` array. If Task 1's artifact is missing at execution time, fail with a clear message rather than creating the root file here.
- Task 3 (USDC spike, Wave 2) consumes this task's Hardhat config to run `contracts/scripts/verify-usdc-eip3009.ts --network arcTestnet`.
- npm deps: `hardhat`, `@nomicfoundation/hardhat-toolbox`, `@nomicfoundation/hardhat-verify`, `@openzeppelin/contracts@^5.0.0`, plus `typescript`, `ts-node`, `dotenv`, `@types/node`, `@types/chai`, `@types/mocha`.

**Edge cases:**
- Arc Testnet RPC may be flaky or rate-limited at smoke time; config must not crash on missing env vars (use `??` defaults, empty `accounts` array if no `DEPLOYER_KEY`). The smoke checks above deliberately use `hardhat config` (offline) rather than `hardhat console --network arcTestnet` (online) to avoid RPC dependency.
- Hardhat compiler download on first run requires network access; CI environments need to allow this or pre-populate the compiler cache.
- `@nomicfoundation/hardhat-verify` for Arc requires custom chain definition; the exact arcscan API endpoint is confirmed in Task 11. The placeholder URLs (`placeholder.example.com`) are intentionally implausible so they fail loudly if a deploy is attempted before Task 11 patches them.
- Hardhat does not support pure ESM config files cleanly with `"type": "module"`; `contracts/` package uses CJS while the root and `packages/middleware/` are ESM. Document this in a top-of-file comment in `hardhat.config.ts`.
- `accounts: []` is valid for compile/test; deploy scripts (Task 11) will fail loudly if `DEPLOYER_KEY` is missing ‚Äî that is correct behavior, do not paper over with a fake key.
- The legacy env-var names `ARC_TESTNET_RPC_URL` and `ARC_TESTNET_PRIVATE_KEY` MUST NOT appear anywhere; they are a known footgun across iteration 1 and are explicitly forbidden by systemic-fix ¬ß3.

**Implementation hints:**
- Reference Hardhat docs for `@nomicfoundation/hardhat-verify` custom-chain shape (`customChains: [{ network: "arcTestnet", chainId: 5042002, urls: { apiURL, browserURL } }]`).
- Pin Hardhat to the latest 2.x line that supports Solidity 0.8.20 and the toolbox version. Caret ranges are fine.
- `@nomicfoundation/hardhat-toolbox` brings in chai, mocha, ethers v6, hardhat-network-helpers, hardhat-gas-reporter, solidity-coverage. Do NOT add these separately ‚Äî version drift risk.
- Solidity optimizer `runs: 200` is the OZ default and matches what their audited contracts assume; do not deviate.
