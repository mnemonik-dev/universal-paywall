## Description

Initialize the `contracts/` Hardhat TypeScript project that will host `PaymentSplitterFactory.sol` + `PaymentVaultImpl.sol` (Wave 2) and the Arc Testnet USDC spike script (Task 3). This task only sets up tooling and configuration ‚Äî no Solidity sources, tests, or deploy scripts are written here. After this task lands, any subsequent contract task can run `cd contracts && npx hardhat compile` against the configured Arc Testnet network and have all required OpenZeppelin 5.x primitives available.

Concrete deliverables:
- `contracts/package.json` declares Hardhat + toolbox + verify plugin + OpenZeppelin 5.x as deps, includes lint/compile/test scripts, and is wired into the npm workspace root from Task 1.
- `contracts/hardhat.config.ts` pins Solidity `^0.8.20` (with optimizer settings sane for prod), defines an `arcTestnet` network (chainId `5042002`, RPC `https://rpc.testnet.arc.network`, accounts read from env), and registers `@nomicfoundation/hardhat-toolbox` + `@nomicfoundation/hardhat-verify`.
- `contracts/tsconfig.json` extends a strict TS config aligned with the rest of the monorepo and includes `hardhat.config.ts`, `deploy/`, `scripts/`, and `test/`.
- Empty placeholder dirs (`contracts/contracts/`, `contracts/test/`, `contracts/scripts/`, `contracts/deploy/`) so `hardhat compile` does not error on missing folders.

Driven by tech-spec D8 (Solidity pragma + OZ 5.x choice), D9 (Hardhat over Foundry), architecture.md "Smart contracts" row, and Wave 1 task list.

## What to do

- Create the `contracts/` directory layout: `contracts/contracts/`, `contracts/test/`, `contracts/scripts/`, `contracts/deploy/`. Add a `.gitkeep` to each so the empty dirs are tracked.
- Write `contracts/package.json`:
  - `"name": "@universal-paywall/contracts"`, `"private": true`, `"type": "module"` consistent with the rest of the workspace (only if compatible with Hardhat; Hardhat TS configs typically use CJS, so use `"type": "commonjs"` and document the deliberate deviation from the root in a code comment at the top of `hardhat.config.ts`).
  - `devDependencies`: `hardhat`, `@nomicfoundation/hardhat-toolbox`, `@nomicfoundation/hardhat-verify`, plus the typescript toolchain (`typescript`, `ts-node`, `@types/node`, `@types/chai`, `@types/mocha`).
  - `dependencies`: `@openzeppelin/contracts@^5.0.0`.
  - Scripts: `compile` (`hardhat compile`), `test` (`hardhat test`), `coverage` (`hardhat coverage`), `lint` (typecheck ‚Äî Solidity linter wiring is out of scope for this task; tech-spec D15/D17 reference slither integration but lint scripts are added later when the contracts exist).
- Write `contracts/hardhat.config.ts` in TypeScript:
  - Imports `@nomicfoundation/hardhat-toolbox` and `@nomicfoundation/hardhat-verify`.
  - `solidity` config pins version `0.8.20` (compatible with pragma `^0.8.20`), enables optimizer (`enabled: true`, `runs: 200`), enables `viaIR: false` for now (toggle on only if size/gas requires).
  - `networks.arcTestnet`: `url: process.env.ARC_RPC_URL ?? "https://rpc.testnet.arc.network"`, `chainId: 5042002`, `accounts: process.env.DEPLOYER_KEY ? [process.env.DEPLOYER_KEY] : []` (empty array is valid for compile/test-only operation; deploy scripts require the env var).
  - `networks.hardhat`: default local network for tests (no extra config needed beyond toolbox defaults).
  - `etherscan` / `verify` section: `apiKey.arcTestnet` from `process.env.ARC_VERIFY_API_KEY ?? ""` and `customChains` entry for arcTestnet pointing at the arcscan testnet explorer URL pair (api + browser). If the exact arcscan URLs are uncertain at the time of writing, leave a `// TODO(task-11): confirm arcscan API + browser URLs` comment and a placeholder ‚Äî verification wiring lives in Task 11. Do not invent fake URLs.
  - Read env vars via `dotenv/config` import at the top, so a `.env` file in `contracts/` is honored. Add `dotenv` to devDependencies if not transitively present.
- Write `contracts/tsconfig.json`:
  - `extends` the root `tsconfig.json` if Task 1 produced one; otherwise inline strict settings (`"strict": true`, `"target": "es2022"`, `"module": "commonjs"` ‚Äî Hardhat requires CJS-flavored TS, `"esModuleInterop": true`, `"resolveJsonModule": true`, `"skipLibCheck": true`).
  - `include`: `["hardhat.config.ts", "deploy/**/*.ts", "scripts/**/*.ts", "test/**/*.ts"]`.
- Confirm npm workspace root (`package.json` at repo root, created by Task 1) lists `contracts` in its `workspaces` array; if not, surface as a deviation rather than silently editing root config (root is owned by Task 1).
- Add `contracts/.env.example` listing `ARC_RPC_URL`, `DEPLOYER_KEY`, `ARC_VERIFY_API_KEY` with placeholder values and a comment that the real `.env` is gitignored (gitleaks-blocked).

## Details

**Files:**
- `contracts/package.json` ‚Äî new. Workspace-private package with Hardhat dev deps + `@openzeppelin/contracts@^5.0.0` dep + `compile`/`test`/`coverage` scripts.
- `contracts/hardhat.config.ts` ‚Äî new. Solidity 0.8.20 pinned, optimizer on, `arcTestnet` network (chainId 5042002, RPC default `https://rpc.testnet.arc.network`, accounts from env), toolbox + verify plugins imported, `dotenv/config` at top, etherscan custom chain stub with TODO for arcscan URLs.
- `contracts/tsconfig.json` ‚Äî new. Strict TS aligned with root, CJS module (Hardhat requirement), includes config + deploy/scripts/test paths.
- `contracts/.env.example` ‚Äî new. Placeholders for `ARC_RPC_URL`, `DEPLOYER_KEY`, `ARC_VERIFY_API_KEY`.
- `contracts/{contracts,test,scripts,deploy}/.gitkeep` ‚Äî empty placeholders so Hardhat sees the folders.

**Dependencies:**
- No task dependencies (Wave 1, parallel with Task 1 and Task 3).
- Task 1 owns the repo-root `package.json` and workspaces declaration; if `contracts` is not listed, surface to the team lead rather than editing root config from this task.
- npm deps: `hardhat`, `@nomicfoundation/hardhat-toolbox`, `@nomicfoundation/hardhat-verify`, `@openzeppelin/contracts@^5.0.0`, plus `typescript`, `ts-node`, `dotenv`, `@types/node`, `@types/chai`, `@types/mocha`.

**Edge cases:**
- Arc Testnet RPC may be flaky or rate-limited; the config must not crash on missing env vars (use `??` defaults, empty `accounts` array if no `DEPLOYER_KEY`).
- Hardhat compiler download on first run requires network access; CI environments need to allow this or pre-populate the compiler cache.
- `@nomicfoundation/hardhat-verify` for Arc may need a custom chain definition that includes the arcscan testnet API endpoint. The exact endpoint is confirmed in Task 11 (deploy); for now leave a `TODO(task-11)` comment with placeholder strings clearly marked `placeholder.example.com` so they fail loudly if used.
- Hardhat does not support pure ESM config files cleanly with `"type": "module"` (paths to TS configs differ); the `contracts/` package therefore uses CJS while the root and `packages/middleware/` are ESM. Document this in a top-of-file comment in `hardhat.config.ts`.
- `accounts: []` is valid for compile/test; deploy scripts (Task 11) will fail loudly if `DEPLOYER_KEY` is missing ‚Äî that is correct behavior, do not paper over with a fake key.

**Implementation hints:**
- Reference Hardhat docs for `@nomicfoundation/hardhat-verify` custom-chain shape (`customChains: [{ network: "arcTestnet", chainId: 5042002, urls: { apiURL, browserURL } }]`).
- Pin Hardhat to the latest 2.x line that supports Solidity 0.8.20 and the toolbox version. Do not pin patch versions tighter than needed ‚Äî caret ranges are fine.
- `@nomicfoundation/hardhat-toolbox` brings in chai, mocha, ethers v6, hardhat-network-helpers, hardhat-gas-reporter, solidity-coverage. No need to add these separately ‚Äî that would risk version drift.
- Solidity optimizer `runs: 200` is the OZ default and matches what their audited contracts assume; do not deviate.
