# Task 5: Contract tests (Foundry)

## Required Skills

Перед выполнением задачи загрузи:
- `/skill:code-writing` — [skills/code-writing/SKILL.md](~/.claude/skills/code-writing/SKILL.md)

## Description

Написать полный набор **Foundry (forge)** тестов в Solidity для `PaymentSplitterFactory` и `PaymentVaultImpl` (контракты из Task 4). Тесты лежат в `contracts/test/*.t.sol`, наследуют `forge-std/Test.sol`, используют `vm.*` cheats. Покрытие: **unit + fuzz + invariant** тесты, ≥95% branch coverage на обоих контрактах через `forge coverage --report lcov`.

Это последняя линия защиты перед security audit (Task 13) и forked-e2e (Task 10). Любая невыявленная ветка в `register()`, `withdraw()`, `setFeeBps()` или в guard-инвариантах vault (D15/D16/D17) — потенциальная уязвимость в money-handling коде. Тесты пишутся **после** реализации Task 4: контракты были спроектированы в tech-spec, реализованы в Task 4 — здесь мы проверяем, что реализация соответствует спеку, и пинаем структурные/property-инварианты, которые должны сохраняться при будущих рефакторингах.

**Foundry-specific преимущества (per addendum §4 T5 + D9):**
- Solidity-native test harness: моки лежат в `contracts/test/mocks/*.sol` и автоматически компилируются Foundry (`test` path is compiled), без отдельного TS-runner'а.
- Fuzz-тесты с встроенным shrinker'ом для feeMath / register-идемпотентности.
- Invariant-тесты с handler-based stateful fuzzing для интегральных инвариантов vault.
- `MockMaliciousTreasury` re-enters `vault.withdraw()` во время fee-transfer — реальная динамическая reentrancy-проверка (не симуляция).

**Каноническая структура (Foundry layout per addendum §1):**
```
contracts/
  foundry.toml
  src/
    PaymentSplitterFactory.sol         # Task 4
    PaymentVaultImpl.sol               # Task 4
  test/
    PaymentSplitterFactory.t.sol       # этой таски
    PaymentVaultImpl.t.sol             # этой таски
    mocks/
      MockUsdcEip3009.sol              # Task 4 — переехал из contracts/contracts/mocks/
      MockMaliciousTreasury.sol        # этой таски
    invariants/
      VaultInvariants.t.sol            # этой таски
  remappings.txt                       # @openzeppelin/contracts/=lib/openzeppelin-contracts/contracts/
```

`MockUsdcEip3009.sol` уже лежит в `contracts/test/mocks/` после Task 4 (перевезён туда в рамках Foundry-миграции). Эта таска добавляет `MockMaliciousTreasury.sol` рядом и тестовые `.t.sol` файлы.

**Канонический конструктор фабрики (iteration-3 §1):**
```solidity
constructor(IERC20 _usdc, address _platformTreasury, uint16 _initialFeeBps)
```
Ровно **3 аргумента**. `vaultImpl` создаётся внутри конструктора (`vaultImpl = address(new PaymentVaultImpl())`). `initialOwner` приходит из `Ownable2Step` base — `msg.sender` деплой-транзакции. `feeBps` — `uint16`, не `uint256`.

## What to do

### PaymentSplitterFactory.t.sol

Контракт `PaymentSplitterFactoryTest is Test` с `setUp()`, который деплоит `MockUsdcEip3009`, фабрику с 3 аргументами `(mockUsdc, treasury, 50)`, и подготавливает несколько dev EOA (через `makeAddr("dev1")`, `vm.addr(privKey)`).

Покрыть следующие группы тестов (ровно по Testing Strategy → Contract tests):

1. **`register()` — happy path и инварианты**
   - `test_Register_DeploysVaultAtPredictedAddress` — `register()` возвращает CREATE2-предсказанный адрес.
   - `test_Register_PopulatesVaultsMapping` — после register: `vaults[developer]` равен задеплоенному адресу.
   - `test_Register_VaultInitializedWithDeveloper` — vault `developer()` == `msg.sender` register-call'а.
   - `test_Register_RevertsAlreadyRegistered` — второй вызов `register()` от того же EOA → revert `AlreadyRegistered`.
   - `test_Register_RevertsWhenPaused` — owner вызвал `pause()`, register → revert `EnforcedPause()` (OZ 5.x).

2. **CREATE2 cross-component invariant (Solidity-side via OZ `Clones`)**
   - `test_Register_PredictedAddressMatchesClonesPredict` — для минимум 3 разных developer EOA: вычислить ожидаемый адрес через `Clones.predictDeterministicAddress(factory.vaultImpl(), salt, address(factory))` где `salt = bytes32(uint256(uint160(developerEoa)))`. Этот адрес должен byte-в-byte совпадать с `factory.computeVaultAddress(developerEoa)`.
   - **Per addendum §4 T5**: используем OZ `Clones.predictDeterministicAddress` напрямую в Solidity тесте — никакого off-chain TS compute не нужно. Это убирает целый класс багов "TS-side формула расходится с on-chain формулой".
   - Инвариант: на нём держится off-chain маршрутизация `payTo` в middleware (D3). Если разъезжается — middleware будет отправлять платежи на адрес, который никогда не задеплоится.

3. **`register()` reentrancy invariant — static analysis, not dynamic (iteration-3 §15, applies per addendum §4 T5)**
   - **Динамический reentrancy-тест исключён**: `PaymentVaultImpl.initialize()` не делает внешних вызовов на developer-EOA, callback-path отсутствует, поэтому реентри-вектор структурно невозможен.
   - **Заменяющая проверка**: `slither --detect reentrancy-eth,reentrancy-no-eth contracts/src/` запускается в CI (см. addendum §1 — `slither.config.json` excludes `test/`). Этот тест НЕ в `.t.sol` файле; он в CI pipeline.
   - **В `.t.sol`**: добавить только NatSpec-комментарий на test-contract: «Reentrancy в register() структурно невозможен (нет external calls до state writes); инвариант пинуется в Task 4 через NatSpec на самом `register()` + в CI через slither.»

4. **`setFeeBps(uint16 bps)`**
   - `test_SetFeeBps_OwnerOnly` — non-owner получает revert `OwnableUnauthorizedAccount(account)` (OZ 5.x).
   - `test_SetFeeBps_RevertsOnTooHigh` — `bps = 1001` → revert `InvalidFeeBps`. Также `bps = type(uint16).max` → revert.
   - `test_SetFeeBps_AcceptsValidValues` — для `bps ∈ {0, 50, 500, 1000}` успешно обновляет состояние.
   - `test_SetFeeBps_EmitsEvent` — `vm.expectEmit(true, true, false, true); emit FeeBpsUpdated(oldBps, newBps);`.

5. **`setPlatformTreasury(address)`**
   - `test_SetPlatformTreasury_OwnerOnly`.
   - `test_SetPlatformTreasury_RevertsOnZero` — revert `ZeroAddress`.
   - `test_SetPlatformTreasury_HappyPathEmitsEvent` — `PlatformTreasuryUpdated(oldTreasury, newTreasury)`.

6. **`Ownable2Step`**
   - `test_Ownable2Step_TransferRequiresAccept` — `transferOwnership(newOwner)` сам по себе не передаёт владение; `setFeeBps` от newOwner до accept → revert.
   - `test_Ownable2Step_AcceptCompletesTransfer` — после `vm.prank(newOwner); factory.acceptOwnership()` старый owner теряет owner-only функции.
   - `test_Ownable2Step_CancelByNewTransfer` — `transferOwnership(newOwner)` затем `transferOwnership(address(0))` (или другой адрес) отменяет pending.

7. **`pause()` / `unpause()`**
   - `test_Pause_OwnerOnly`.
   - `test_Pause_BlocksRegister` — после `pause()`: `paused() == true`, `register()` revert'ит `EnforcedPause()`.
   - `test_Unpause_RestoresRegister` — после `unpause()`: `register()` снова работает.

8. **Constructor reverts (3-arg signature, iteration-3 §1)**
   - `test_Constructor_RevertsOnZeroUsdc` — `_usdc == address(0)` → revert `ZeroAddress`.
   - `test_Constructor_RevertsOnZeroTreasury` — `_treasury == address(0)` → revert `ZeroAddress`.
   - `test_Constructor_RevertsOnTooHighFee` — `_feeBps > 1000` → revert `InvalidFeeBps`.
   - `test_Constructor_HappyPathReadsAllState` — deployed с валидными `(usdc, treasury, 50)` — `usdc()`, `platformTreasury()`, `feeBps()`, `vaultImpl()`, `owner()` все читаются корректно.
   - `test_Constructor_VaultImplDeployedInternally` — `vaultImpl()` возвращает non-zero адрес (фабрика задеплоила impl внутри конструктора).

9. **`VaultDeployed` event + edge cases**
   - `test_Register_EmitsVaultDeployed` — event arg `vault` равен предсказанному CREATE2 адресу; event arg `developer` равен msg.sender.
   - `test_Vaults_UnregisteredReturnsZero` — `vaults[unregistered_address]` возвращает `address(0)`.
   - `test_Register_TwoDevsProduceDistinctVaults` — два разных EOA → distinct vault addresses (через `vm.prank` + два register call'а).

### PaymentVaultImpl.t.sol

Контракт `PaymentVaultImplTest is Test` с `setUp()`, который деплоит фабрику, registers `dev1`, и кэширует адрес `vault = factory.vaults(dev1)`. Все тесты через **clone**, развёрнутый из factory, кроме явных D15 тестов на impl.

1. **`initialize` single-call**
   - `test_Initialize_FirstCallSucceeds` — это проверяется тем фактом, что `setUp()` уже вызвал `factory.register()`.
   - `test_Initialize_RevertsOnSecondCall` — прямой `IPaymentVault(vault).initialize(otherDev)` после деплоя → revert `InvalidInitialization()` (OZ 5.x).

2. **D15: `_disableInitializers()` на impl**
   - `test_D15_DirectInitializeOnImplReverts` — получить `factory.vaultImpl()` (это сам impl, не клон); прямой вызов `IPaymentVault(vaultImpl).initialize(anyAddr)` → revert `InvalidInitialization()`.
   - Канонический hijack-vector для cloneable контрактов; явный отдельный тест.

3. **D16: нет `receive()` / `fallback()`**
   - `test_D16_NativeTransferReverts` — `vm.deal(address(this), 1 ether); (bool ok,) = vault.call{value: 1}(""); assertFalse(ok);`. Тестируем EVM-инвариант "нет payable точек входа".

4. **D17: ABI-assertion — нет setters для `developer` / `factory`**
   - В Foundry самый чистый путь: попытаться вызвать `setDeveloper(address)` и `setFactory(address)` через low-level `call` с rawly-encoded селектором; assertFalse(ok), потому что у vault нет этих функций (revert на dispatch).
   - `test_D17_NoSetDeveloperSelector` — `bytes4 sel = bytes4(keccak256("setDeveloper(address)")); (bool ok,) = vault.call(abi.encodeWithSelector(sel, address(0xBEEF))); assertFalse(ok);`.
   - `test_D17_NoSetFactorySelector` — аналогично для `setFactory(address)`.
   - Дополнительно: assert `developer != address(0)` после initialize (state-positive invariant) — это попадает и в invariant-тест ниже.

5. **`withdraw` — happy path и базовые ветки (developer-first transfer order)**
   - Подготовка: `mockUsdc.mint(vault, gross)`.
   - `test_Withdraw_RevertsForNonDeveloper` — `vm.prank(notDev); IPaymentVault(vault).withdraw(); → vm.expectRevert(NotDeveloper.selector)`.
   - `test_Withdraw_RevertsOnZeroBalance` — `vm.prank(dev1); vault.withdraw(); → vm.expectRevert(NoBalance.selector)`.
   - `test_Withdraw_HappyPathDeveloperFirst` — `gross = 1000 micro`, `feeBps = 50`. Snapshots: `devBalBefore = mockUsdc.balanceOf(dev1); treasuryBalBefore = mockUsdc.balanceOf(treasury);`. После `vault.withdraw()`:
     - `dev1` получает `gross - fee = 995`,
     - `treasury` получает `fee = 5`,
     - `vault` balance == 0,
     - событие `Withdrawal(dev1, 1000, 5)` emit'ится.
   - **Transfer order (per systemic-fix iteration-2 §7):** developer FIRST, treasury SECOND. Это пинается косвенно через `MockMaliciousTreasury` тест ниже (если treasury был бы первым, malicious treasury получил бы шанс re-enter ДО того как developer получил свои деньги). Дополнительно — комментарий в тесте.

6. **Fee edge cases (parametrized)**
   - `test_Withdraw_FeeBps0_NoSecondTransfer` — `feeBps == 0`, `gross == 1000` → `fee == 0`. Treasury балaнс не изменяется.
   - `test_Withdraw_FeeBps1000_Splits90_10` — `gross = 1000`, `feeBps = 1000` → dev получает 900, treasury получает 100.
   - `test_Withdraw_DustGrossTruncatesFee` — `gross == 1` micro-USDC, `feeBps == 50` → `fee == 0`, весь 1 unit dev.
   - `test_Withdraw_Boundary199` — `gross == 199`, `feeBps == 50` → `fee == 0`.
   - `test_Withdraw_Boundary200` — `gross == 200`, `feeBps == 50` → `fee == 1`, dev `199`.

7. **Fee-snapshot semantics (current behavior pinning)**
   - `test_Withdraw_UsesCurrentFeeBpsAtWithdrawTime` — deploy vault, `feeBps == 50`. Fund vault на 1000 micro. Owner вызывает `factory.setFeeBps(100)`. Fund vault на ещё 1000 micro (теперь vault balance == 2000). Developer вызывает `withdraw()`. Ожидаемое: `fee = 2000 * 100 / 10000 = 20`, dev получает 1980.
   - Документировано: fee читается at withdraw time, не snapshot'ится per payment. Тест pin'ит текущее поведение, чтобы случайное изменение в коде (snapshot per-payment) сразу провалило тест.

8. **Reentrancy через `MockMaliciousTreasury` (dynamic, addendum §4 T5)**
   - **Vector**: vault шлёт fee `platformTreasury` ВТОРЫМ (после developer). Если treasury — это контракт `MockMaliciousTreasury`, его callback при получении USDC пытается re-enter `vault.withdraw()`.
   - **Setup**: deploy `MockMaliciousTreasury`, deploy новую фабрику с `treasury = address(maliciousTreasury)`, register `dev1`, fund vault на 1000 micro. `maliciousTreasury.setTarget(vault)`. От `dev1`: `vault.withdraw()`.
   - **Ожидаемое**: re-entry blocked OZ `ReentrancyGuard` (revert `ReentrancyGuardReentrantCall()`). Внешний вызов `withdraw()` revert'ит целиком; state не изменился (vault balance после revert остаётся 1000).
   - `test_Withdraw_ReentrancyBlocked_ViaMaliciousTreasury` — `vm.prank(dev1); vm.expectRevert(...); vault.withdraw();` + asserts на balance после revert.

9. **Withdraw работает при `paused()` factory**
   - `test_Withdraw_WorksWhenFactoryPaused` — register dev, fund vault, owner вызывает `factory.pause()`. `vm.prank(dev1); vault.withdraw();` → success. Developers никогда не блокируются от снятия (D12).

### Fuzz tests (per addendum §4 T5)

В `PaymentSplitterFactory.t.sol` ИЛИ `PaymentVaultImpl.t.sol` (выбрать релевантный файл; рекомендуется vault — fee math, factory — register):

1. **`testFuzz_FeeMath(uint256 gross, uint16 feeBps)`** (vault)
   - Bound: `gross ∈ [0, 1e18]` (разумный диапазон), `feeBps ∈ [0, 1000]`.
   - Setup: `factory.setFeeBps(feeBps)`, `mockUsdc.mint(vault, gross)`.
   - Если `gross == 0`: `vm.expectRevert(NoBalance.selector); vault.withdraw();`.
   - Если `gross > 0`: после `withdraw()`:
     - `fee = gross * feeBps / 10000`,
     - `dev gets gross - fee`,
     - `assert fee + net == gross` (no overflow, no negative),
     - `assert fee <= gross`,
     - `assert vault.balance == 0`.

2. **`testFuzz_RegisterIdempotent(address dev)`** (factory)
   - Bound: `vm.assume(dev != address(0));` (избежать тривиального reverts).
   - `vm.prank(dev); factory.register();` → success.
   - `vm.prank(dev); vm.expectRevert(AlreadyRegistered.selector); factory.register();`.
   - Property: для любого валидного `dev`, второй вызов `register()` от этого же EOA всегда revert'ит `AlreadyRegistered`, независимо от значения адреса.

### Invariant tests (per addendum §4 T5)

`contracts/test/invariants/VaultInvariants.t.sol` — Foundry stateful fuzzing с handler:

```solidity
contract VaultInvariants is Test {
    PaymentSplitterFactory factory;
    PaymentVaultImpl vault;
    MockUsdcEip3009 mockUsdc;
    address dev1;
    VaultHandler handler;

    function setUp() public {
        // deploy factory, mockUsdc, register dev1
        // deploy handler with refs to vault + mockUsdc
        // targetContract(address(handler));
    }

    /// @dev Vault USDC balance == sum(mints) - sum(withdrawals)
    function invariant_VaultBalanceIntegrity() public view {
        assertEq(
            mockUsdc.balanceOf(address(vault)),
            handler.totalMinted() - handler.totalWithdrawn()
        );
    }

    /// @dev Factory feeBps never exceeds 1000 (uint16 cap)
    function invariant_FeeBpsBounded() public view {
        assertLe(factory.feeBps(), 1000);
    }

    /// @dev Vault developer is never zero after initialization
    function invariant_DeveloperNonZero() public view {
        assertNotEq(IPaymentVault(address(vault)).developer(), address(0));
    }
}

contract VaultHandler is Test {
    // Tracks state for invariant checks.
    // Functions: depositToVault(uint256), withdrawFromVault(), setFeeBps(uint16).
    // Bounds inputs sensibly so the fuzzer hits reachable states.
}
```

Конкретное определение `VaultHandler` уточнить при имплементации — ключевое: bound входы так, чтобы fuzzer попадал в reachable states (не тривиально-revert'ящиеся вызовы), и tracking'и `totalMinted`/`totalWithdrawn` совпадали с реальностью.

  ```solidity
  function testFuzz_FeeMath(uint256 gross, uint16 feeBps) public {
      gross = bound(gross, 1, 1e18);
      feeBps = uint16(bound(feeBps, 0, 1000));

      vm.prank(address(this)); // test contract is factory owner
      factory.setFeeBps(feeBps);
      mockUsdc.mint(address(vault), gross);

      uint256 devBalBefore = mockUsdc.balanceOf(dev1);
      uint256 treasuryBalBefore = mockUsdc.balanceOf(treasury);

      vm.prank(dev1);
      IPaymentVault(address(vault)).withdraw();

      uint256 fee = (gross * feeBps) / 10000;
      uint256 net = gross - fee;

      assertEq(mockUsdc.balanceOf(dev1) - devBalBefore, net, "dev net mismatch");
      assertEq(mockUsdc.balanceOf(treasury) - treasuryBalBefore, fee, "treasury fee mismatch");
      assertEq(net + fee, gross, "split sum != gross");
      assertEq(mockUsdc.balanceOf(address(vault)), 0, "vault not drained");
  }
  ```
