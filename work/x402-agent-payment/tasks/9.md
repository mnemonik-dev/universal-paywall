# Task 9: Middleware unit tests (incl. adapter unit tests)

## Required Skills

Before starting, load:
- `/skill:code-writing` ‚Äî [skills/code-writing/SKILL.md](~/.claude/skills/code-writing/SKILL.md)

## Description

Implement the complete unit test suite for `@universal-paywall/middleware` per tech-spec "Testing Strategy ‚Üí Unit tests" (‚â•85% line coverage). The suite must validate every pure-module and orchestration invariant defined in Waves 4-5 (Tasks 6, 7, 8): the x402 codec and 4 KB header cap, network-id normalization (CAIP-2 ‚Üî alias), EIP-712 verify with full tamper matrix, the synchronous NonceStore semantics and TTL/cap eviction, the 7-reason settlement failure taxonomy with replay-store retention, OpaqueRelayerKey defense-in-depth across every serialization surface, the canonical error-response bodies, the core orchestrator's full happy/sad paths, USD price parsing edge cases, the D14 startup chainId pin, the D18 SecurityLogger event catalog with redaction, the factory-state cache with TTL/RPC-error semantics, and the framework-agnostic adapter contracts (Node http exception propagation + Fastify `preHandler` lifecycle).

This task is the safety net for the entire middleware. Every Decision that has a runtime invariant (D1, D5, D13, D14, D18) and every Risk row with a "test asserts" mitigation MUST be covered by a named test here. The 402 body builder is additionally validated against the vendored x402 v1 JSON Schema using ajv so any future drift from the wire format is caught at unit-test time.

## What to do

1. Vendor the x402 v1 schema as a fixture: download/transcribe the schema from the x402 spec into `packages/middleware/__tests__/fixtures/x402-v1.schema.json` (covers `PaymentRequirements`, `PaymentPayload`, `ExactEvmPayload`, the 402 body envelope, and the X-PAYMENT-RESPONSE shape).
2. Set up `vitest` with `--coverage` (v8 provider) configured to enforce ‚â•85% line coverage on `packages/middleware/src/`. Coverage thresholds go in `vitest.config.ts` (or `package.json` `vitest` field). Build pipeline must fail when below threshold.
3. Add an `__tests__/helpers/` directory with shared test utilities: viem-based EIP-712 signer factory (so each verify test mints a fresh valid signature and then mutates one field), a `FakeTimers` wrapper for TTL tests, a `MockPublicClient`/`MockWalletClient` for settle and factory-state tests, and a header-encoder helper that produces both well-formed and malformed X-PAYMENT bytes.
4. Implement one test file per src module, plus dedicated orchestration and adapter files:
   - `x402.test.ts` ‚Äî build/decode/encode + ajv schema validation + 4 KB cap + strict-shape rejection.
   - `networks.test.ts` ‚Äî CAIP-2 ‚Üî alias normalization round-trips, unknown-network handling.
   - `verify.test.ts` ‚Äî full tamper matrix (chainId, verifyingContract, domain.name, domain.version, to, value, validAfter, validBefore, network mismatch) plus valid-signature happy path.
   - `replay-store.test.ts` ‚Äî synchronous has+insert (TOCTOU), TTL eviction on every `has`, 100k cap eviction by oldest `validBefore`, double `(from, nonce)` rejection.
   - `settle.test.ts` ‚Äî each of the 7 settlement_failed reasons (rpc_timeout, rpc_5xx, gas_estimate_revert, mine_timeout, receipt_reverted, relayer_no_balance, authorization_already_used_onchain) plus replay-store-retention-on-failure invariant.
   - `relayer-key.test.ts` ‚Äî every serialization surface in D13 (JSON.stringify, toString, util.inspect with the custom symbol, pino, winston, structuredClone), brand-symbol redaction, error-stack 64-hex scrub, non-enumerable field.
   - `errors.test.ts` ‚Äî each error reason produces a canonical body validated against the schema.
   - `core.test.ts` ‚Äî orchestrator: missing header ‚Üí 402, happy path ‚Üí passthrough + response headers, settle failure ‚Üí 402 with the failure reason, network mismatch ‚Üí 402, paused factory ‚Üí 402, vault_not_deployed ‚Üí 402; D14 chainId pin on first request; D18 SecurityLogger fires the right event for each branch.
   - `price.test.ts` ‚Äî USD price parsing edge cases: `'0.01'` ‚Üí `10000n`, `'1.5'` ‚Üí `1500000n`, rejections for `'1.2345678'`, `'abc'`, `''`, `'-1'`, `'0'`, `'1e2'`, `' 1 '`.
   - `factory-cache.test.ts` ‚Äî TTL 5 s behavior, fresh registration invisible until expiry, RPC error during cache fill returns last-good if non-stale else propagates as `settlement_failed.reason = "rpc_5xx"`.
   - `security-logger.test.ts` ‚Äî each of the 10 named events fires when triggered, payload shape per D18 table (payerHash = keccak256(from)[0..8], not raw address), default no-op produces nothing, try/catch swallowing of a throwing logger.
   - `adapters/node-http.test.ts` ‚Äî handler exception propagation through `withPaywall`, response-header flushing order, 402 path, 200 passthrough path.
   - `adapters/fastify.test.ts` ‚Äî `preHandler` integration, 402 reply on absent X-PAYMENT, Fastify reply chaining preserved on 200, error propagation.
5. Each test must follow TDD ordering: write the test, observe failure against the existing src from Task 8, then verify it passes against the implemented module. If a test reveals a defect in Task 6/7/8 code, file a deviation note and fix the source rather than relaxing the test.
6. Run `npm test --workspace=packages/middleware -- --coverage` and confirm ‚â•85% line coverage. Investigate every uncovered line ‚Äî either add a test or document why it is unreachable.
