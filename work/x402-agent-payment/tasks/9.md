## Description

Implement the complete unit test suite for `@universal-paywall/middleware` per tech-spec "Testing Strategy → Unit tests" (≥85% line coverage). The suite must validate every pure-module and orchestration invariant defined in Waves 5-7 (Tasks 6, 7, 8): the x402 codec and 4 KB header cap, network-id normalization (CAIP-2 ↔ alias), EIP-712 verify with full tamper matrix (return-value taxonomy only per addendum §2), USD price parsing edge cases including `'0'` throwing per addendum §4 (parseUsdPrice lives in `x402.ts` per iteration-2 §6), the synchronous NonceStore semantics and TTL/cap eviction, the 7-reason settlement failure taxonomy (return-value taxonomy only per addendum §2) with replay-store retention, OpaqueRelayerKey defense-in-depth across every serialization surface, the canonical error-response bodies and reason strings (iteration-2 §4), the core orchestrator's full happy/sad paths, the D14 startup chainId pin, **the complete D18 SecurityLogger event catalog asserted exclusively from `core.test.ts` (per addendum §2: `core.ts` is the only owner of `logger.securityEvent(...)` calls; `verify.ts` and `settle.ts` do not import the logger)** with `payerHash` in canonical 10-char form `'0x' + keccak256(from).slice(2, 10)` per addendum §5, the factory-state cache with TTL/RPC-error semantics (cache lives inside `core.ts` per iteration-2 §5), and the framework-agnostic adapter contracts (Node http exception propagation + Fastify `preHandler` lifecycle).

This task is the safety net for the entire middleware. Every Decision that has a runtime invariant (D1, D5, D13, D14, D18) and every Risk row with a "test asserts" mitigation MUST be covered by a named test here. The 402 body builder is additionally validated against the vendored x402 v1 JSON Schema using ajv so any future drift from the wire format is caught at unit-test time.

**T9 also owns the x402 v1 schema fixture at `packages/middleware/src/__tests__/fixtures/x402-v1.schema.json`.** T10 (forked + Arc Testnet e2e, Wave 9) consumes it.
